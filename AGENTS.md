# 프로젝트 작업 규칙

## 소스 탐색 규칙 (필수)

- 기존(as-is) 소스 중, **폴더명이 `_`(언더바)로 시작하는 폴더는 탐색 대상에서 제외**한다.
  (백업·구버전·실험/폐기 코드이며, 현행 산출물이 아니다. 예: `_backup_ui5_*`,
  `_module_dialog_pattern`, `_ServerList_ui5` 등.)
- 검증·참조·변환 근거를 찾을 때 이 폴더들의 내용을 현행으로 인용하지 않는다.

## UX / 화면 / 컨버전 작업 규칙 (필수)

이 프로젝트의 UX·화면·UI5→HTML5 컨버전 표준의 **단일 출처(SSOT)는 `.analy/` 문서 세트**다.
화면, UI, 레이아웃, 모달/팝업, 트리, 폼, 입력칸, 그리드 등 **화면 요소를 만들거나
U4A WS → HTML로 변환하는 모든 작업**은 아래 절차를 따른다.

### 1. 시작 전 반드시 읽기
- `.analy/00_개요_및_컨버전전략.md` — 문서 인덱스로 어떤 문서를 봐야 하는지 먼저 파악한다.
- `.analy/13_AI_작업지시_가이드.md` — AI 작업 방식·지시 템플릿·가드레일. 이 문서의 원칙을 그대로 따른다.
- **`.analy/16_공통_화면UX_표준.md`** — 모든 화면 공통 표준(Bootstrap 베이스, 모달 헤더/푸터 48px,
  드래그·리사이즈, 트리=서버리스트). 화면 작업 시 항상 함께 적용한다.
- 작업 대상에 해당하는 **기능 문서 + `.analy/12_테마_컨버전_전략.md`(테마)**를 끝까지 읽고 시작한다.
  - 입력칸(input) 관련: `.analy/15_공통_입력UX_가이드.md`
  - 팝업/모달: `.analy/06_팝업.md`
  - 로그인/서버리스트(트리 포함): `.analy/02_로그인_서버리스트.md`
  - 디자인 영역: `.analy/05_디자인영역.md`
  - 그 외 매핑은 13번 문서 3절의 표를 참고한다.

### 2. 진행 방식 (13번 문서 원칙)
- **한 번에 작업 단위 하나.** "전부 컨버전"이 아니라 해당 문서의 `AI 컨버전 실행 사양` 절 하나씩.
- 각 문서의 작업 순서를 그 순서대로 수행하고, **산출물 트리에 명시된 파일만** 만들거나 고친다.
- 모든 설계 결정은 **문서의 절 번호로 근거**를 댄다. 문서에 없으면 **추측하지 말고 사용자에게 묻는다.**
- 모양/동작은 화면마다 새로 만들지 말고 **공통 자산(`shell.css`, `u4a-ui.js`, `tokens.css`)을 소비**한다.
- 색은 `theme/tokens.css`의 의미 토큰만 사용한다. **하드코딩(hex) 금지**, 문구는 메시지 키 사용.
- **반드시 반응형** — 고정 px 폭 금지(12번 7장).

### 3. 완료 보고

- 각 단계 수용 기준 통과/실패 표
- 12번 6.6 체크리스트(A~J) 점검 결과
- 문서와 어긋났거나 추측한 부분 목록
