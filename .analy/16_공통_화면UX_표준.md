# 16. 공통 화면 UX 표준

> 모든 화면에 **공통으로 적용**되는 UX 표준(전역 규칙)이다. 특정 기능 문서(02~08)나
> 입력 가이드(15)보다 상위의 "모든 화면 공통" 기준이며, 화면을 새로 만들거나
> U4A WS → HTML5 로 변환할 때 항상 함께 적용한다.
> 관련: [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md)(테마 토큰), [15_공통_입력UX_가이드.md](15_공통_입력UX_가이드.md)(입력칸), [06_팝업.md](06_팝업.md)(팝업 카탈로그), [02_로그인_서버리스트.md](02_로그인_서버리스트.md)(트리 기준)

## 0. TL;DR

1. 모든 화면은 **Bootstrap** 을 기준(베이스) 프레임워크로 사용한다.
2. 색·간격 등 **테마는 `tokens.css` 의미 토큰**으로 입힌다 (Bootstrap CSS 변수에 매핑). 하드코딩 금지.
3. 모든 메시지 박스·모달 팝업의 **헤더/푸터 높이는 각각 48px**.
4. 모달 팝업은 **드래그(이동) + 리사이즈(크기 조절)** 가능해야 한다.
5. 트리는 **공통 베이스 트리 `U4AUI.createTree`(`u4a-ui.js`)** 를 소비한다. 화면마다 재귀 렌더러를 새로 만들지 않는다(소스 단일화).
6. 스플릿바(분할 리사이저)는 **공통 골격 `.u4a-splitter*`(`shell.css`) + 인접 재분배 드래그**를 소비하고, 모양은 **서버리스트 기준(가운데 그립 핸들)** 으로 통일한다.
7. 화면 문구는 **전부 SQLite 메시지 클래스 키 + 공통 추출 함수**로 표시한다(하드코딩 금지). 기존 키가 없는 신규 텍스트는 **임의 생성·임의 키 금지**, 모았다가 **신규 항목만 목록으로 제출** → 사용자가 번역·DB 입력·재실행.
8. 목록/그리드는 **공통 `.u4a-table`** 을 소비하고, **컬럼 헤더 정렬/필터 메뉴는 서버리스트 기준**(필터 input → 오름/내림 정렬 → 필터 초기화)으로 통일한다.

## 1. 기준 프레임워크 — Bootstrap

- **모든 화면은 Bootstrap 을 베이스로 사용한다.** 레이아웃·그리드·버튼·폼 등 표준 컴포넌트는
  Bootstrap 컴포넌트/클래스를 우선 소비하고, 화면마다 새로 만들지 않는다.
- **테마는 토큰으로 입힌다.** `theme/tokens.css` 의 의미 토큰(색/간격)을 **Bootstrap CSS 변수
  (`--bs-*`)에 매핑**하여 적용한다. 색을 직접 hex 로 박지 않는다(드리프트 금지). 테마 단일 출처는
  [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md).
- **공통 컴포넌트는 Bootstrap "위에" 얹는다.** 입력칸의 `.u4a-field`(clear/스피너 제거 등,
  [15번](15_공통_입력UX_가이드.md))과 `shell.css` 의 셸 스타일은 Bootstrap 베이스 위에서 동작하는
  프로젝트 공통 자산으로 유지한다. Bootstrap 과 겹치면 프로젝트 공통 컴포넌트 규칙을 우선한다.

> 주의: `.analy` 다른 문서에서 말하는 "bootstrap"은 대개 제거 대상인 **UI5 로더(`sap-ui-bootstrap`)**
> 를 뜻한다. 본 문서의 "Bootstrap"은 **CSS 프레임워크 Bootstrap** 을 가리키며 서로 다른 개념이다.

## 2. 메시지 박스 / 모달 팝업

### 2.1 헤더·푸터 높이
- 모든 메시지 박스와 모달 팝업의 **헤더(header) 높이는 48px**.
- 모든 메시지 박스와 모달 팝업의 **푸터(footer) 높이는 48px**.
- 본문(body)은 헤더/푸터를 제외한 나머지 영역을 차지한다.

### 2.2 드래그 + 리사이즈
- 모달 팝업은 **헤더를 잡고 드래그하여 이동**할 수 있어야 한다. → 전역 자동(`u4a-ui.js _installGlobalDialogDrag`, document 위임 1벌). 클램프: 좌·우·하=뷰포트 안, 위=`.u4a-titlebar` 하단(`_topChromeBottom`). 새 팝업은 `.u4a-dialog`+`.u4a-dialog__header` 구조만 따르면 배선 불필요. **헤더 hover 시 `cursor:move`**(드래그 affordance) — `.u4a-dialog__header` 공통(shell.css), 헤더 안 닫기 X·액션 버튼은 `.u4a-btn-icon/.u4a-btn` 의 pointer 가 이김.
- 모달 팝업은 **우하단 grip(`U4AUI.makeDialogResizable`)으로 리사이즈** 할 수 있어야 한다(좌상단 anchor). ⚠️ CSS 네이티브 `resize:both` 금지(showModal `margin:auto` 라 가운데 기준으로 커짐) — 공통 grip 단일.
- **헤더 더블클릭 = 화면 중앙 복귀**(`makeDialogRecenter`, 드래그 인라인 위치 비움).
- **창 리사이즈 재클램프 (필수, §4.3 스플릿바와 동일 사상)**: 드래그/리사이즈는 다이얼로그에 **인라인 px**(`position:fixed`+`left/top/width/height`)를 박는다. 한번 박히면 창 크기가 변해도 안 따라오므로, **최대화 상태에서 키우거나 옮긴 팝업이 창 restore 후 작은 창을 넘쳐 오른쪽·아래가 잘린다.** → `_installGlobalDialogDrag` 가 **`window resize` 리스너 1벌**로 인라인 px 가 박힌 **열린** `.u4a-dialog` 를 뷰포트 안으로 클램프(`width/height ≤ vw/vh-16`, `left/top` 클램프, 위=`_minTop`). 인라인이 없는(=CSS `vw/vh`+네이티브 중앙정렬) 다이얼로그는 손대지 않는다. (2026-06-23 추가 — 전 `.u4a-dialog` 공통, 새 팝업 배선 불필요.)
- 팝업 카탈로그와 개별 교체 방식은 [06_팝업.md](06_팝업.md)를 따른다.

### 2.3 진행/대기 안내 모달 (공통 일러스트 다이얼로그)
- SAPGUI 실행(Controller 클래스) 등 **응답 대기 안내**는 화면마다 만들지 말고 **공통 `oAPP.common.fnIllustMsgDialogOpen(opts)` 1곳**을 소비한다(`ws_common.js`). 네이티브 `<dialog>` 의 **헤더/푸터 없는 중앙 일러스트 카드**(제목+설명) — §2.1 헤더/푸터 48px 규칙의 예외다. ESC 닫힘 차단.
- 열기/닫기는 IPC `if-browser-interconnection`(PRCCD `01`=open / `02`=close)으로 **같은 client·SYSID 창에 브로드캐스트**(같은 dev 서버를 보는 여러 창이 함께 표시/해제). 호출부는 [07_JS유틸_워커_IPC.md](07_JS유틸_워커_IPC.md) §3-2 참조.
- 일러스트는 **`www/svg/satellite-antenna-loading-animated.svg`** 를 **`<img>` 로 로드**한다. ⚠️ 인라인 금지 — 이 SVG 는 내부 `<style>`(generic `.dish`/`.scan`/`.cross` 등 클래스·keyframe)을 갖고 있어 HTML 에 인라인하면 그 스타일이 **문서 전역으로 새어** 충돌한다(`<img>` 로 로드하면 이미지 문서에 샌드박스되고 CSS 애니메이션은 그대로 재생).
- 카드/텍스트는 **테마 토큰**(`--surface-raised`/`--text`/`--line`) 소비, 일러스트는 외부 SVG 의 고정 블루(원본 SAP IllustratedMessage 처럼 **테마 무관**). 백드롭은 **옅은 중립**(`rgba(15,18,28,.32)` + 약블러) — 과거 `rgba(2,6,12,.55)` 다크 레이더는 너무 어두워 폐기(2026-06-23).
- ⚠️ **일러스트 SVG 배경은 반드시 투명**이어야 한다. 카드는 테마색(다크=어두움)인데 SVG 에 불투명 `<rect fill="#fff">` 등 배경이 박혀 있으면 다크 테마에서 **흰 블록**으로 떠 어색하다(2026-06-23 사례). 해당 SVG 의 흰 배경 사각형 + 라이트 전용 바닥 타원(`#dff5ff`/`#e9f7ff`)을 제거해 투명화 → 라이트/다크 양쪽에서 카드 위에 자연스럽게 얹힌다. 새 일러스트 추가 시에도 동일(불투명 배경 금지).
- ⚠️ 에셋 경로는 **호스트 문서(`index.html` = `www/ws30/ws10_20/`) 기준 `../../svg/...`** 로 해석한다. `ws_common.js` 는 `<script src>` 가 아니라 **ajax+eval 로 로드**(`library-preload.js`)되어 **DOM 에 스크립트 태그가 없으므로**, 에셋 URL 을 스크립트 src 에서 찾지 말고 `new URL(rel, window.location.href)` 로 만든다.

### 2.4 토스트 (MessageToast 대체) — 공통 `.u4a-toast` (필수)
- **단일 출처 = `theme/shell.css` 의 `.u4a-toast`** (색은 `bootstrap-skin.css`). 화면/팝업마다 토스트 스타일을 새로 만들지 말 것.
- **위치는 반드시 화면 정중앙(상하·좌우 가운데).** `position:fixed; left/top:50%; transform:translate(-50%,-50%)` — shell.css 가 강제. **하단/모서리 토스트 금지**(전 화면 공통 UX). `pointer-events:none` 라 클릭 통과.
- **표시 방식**: 싱글톤 `<div class="u4a-toast" role="alert">` 1개를 만들어(또는 재사용) `textContent` 세팅 후 `dataset.show="true"`, **3초 뒤 `data-show="false"`** 로 자동 숨김(fade 는 CSS). 메인 셸은 `showMessage`/`setMessageToast`(resources/index.js)가, BrowserWindow 팝업은 화면 로컬 `_toast()`(ServerList/optionPopup/editorPopup 동일 패턴)가 이 컴포넌트를 소비.
- 문구는 **메시지 클래스 키만**(§5 — 임의 완료문구 금지). 예: 저장 = `MSG_WS 330`(&1 = `CL_WS_COMMON D23`).

### 2.5 다이얼로그 헤더 "선두 상태 아이콘" 색 (필수)
- 모든 팝업 헤더의 **선두 상태 아이콘(`.u4a-dialog__header > i` 직계, 또는 별창 타이틀바 선두 아이콘)은 테마 accent 색**으로 통일한다. **단일 출처 = `shell.css` `.u4a-dialog__header > i { color: var(--accent) }`(기본).** 일반 팝업은 **아무 것도 안 해도 accent** — 화면별로 색을 따로 주거나 `data-type` 을 달지 않는다.
- **예외 = 메시지박스만**: `fnConfirmBox`/`showMessage` 가 헤더에 `data-type`(S/E/W/I/C)을 세팅 → 의미색 override(E=`--state-error` 빨강 / W=`--state-warning` 주황 / S=`--state-success` 초록 / I·C=accent=기본과 동일). 즉 **data-type 은 "메시지 타입 의미색"용이지 "accent 색 얻기"용이 아니다.**
- ⚠️ 닫기 X·액션 버튼 속 아이콘(`button > i`)은 **직계(`> i`)가 아니라서 제외**된다 — 의미색이 X 버튼까지 칠해지면 안 되므로 `> i` 직계 셀렉터를 반드시 지킨다.
- **2026-06-24 전수 통일**: 전엔 기본이 `--icon-muted`(무색)라, 다수 팝업이 accent 를 얻으려 `data-type="I"` 를 따로 달았고(client_editor·ServerInfo·SysNoti·SelectBrowser·Cts·AppF4·AppCopy·createEvent·createApp), 안 단 일부(USP Create·WS20 Insert/MovePosition/ChangeLayout·ServerList 폼·Login 연결·별창 WinHide)는 muted 로 떠 **불일치**했다. → 기본을 accent 로 바꿔 전부 통일 + 중복 `data-type="I"` 제거(메시지박스 빌더만 동적 data-type 유지). 별창 타이틀바 선두 아이콘(`.u4aWinHide__icon` 등)도 accent.
- ⚠️ **별창 타이틀바 선두 = 화면 성격 따라** — 에디터·매니저류 일반 창은 **U4A 로고**(`.u4a-titlebar__logo`, CSS/JS/HTML Editor·OTR Manager·옵션창 동일), 알림/경고성 창만 의미 아이콘(errMsg=triangle-exclamation, WinHide=eye-slash). **임의로 메뉴 아이콘으로 바꾸지 말 것** — 레퍼런스(editorPopup)가 로고면 로고.

### 2.6 별도 BrowserWindow 창 — 첫 페인트 흰 플래시 방지 + 등장 (필수)
별도 창(`REMOTE.BrowserWindow`)으로 뜨는 팝업(에디터·옵션·OTR Manager·에러메시지 등)은 인앱 모달(`.u4a-dialog`)과 **다른 트랙**이다. 다크/유색 테마에서 **창이 뜰 때 흰 배경이 번쩍였다가 테마가 입혀지는 현상**을 다음으로 막는다. 레퍼런스 = `editorPopup`(CSS/JS/HTML 에디터).

- **네이티브 창 opacity 페이드 금지.** opener 에서 `oBrowserOptions.opacity = 0.0` + 페이지에서 `WSUTIL.setBrowserOpacity()`(OS 레벨 0→1 페이드)를 **쓰지 않는다** — OS 리컴포짓이라 무겁고, 그 사이 흰 배경이 보인다. 대신:
  - opener: `oBrowserOptions.backgroundColor = oThemeInfo.BGCOL`(창을 처음부터 테마색 불투명) + `show:false`. 위치(`setParentCenterBounds`)는 `ready-to-show` 에서 잡고, **페이지가 로드되면 `CURRWIN.show()` 로 표시**한다.
- **첫 페인트 전 동기 테마 적용.** 페이지 `<head>` 에서 theme-api 로드 직후, opener 가 넘긴 쿼리 `BGCOL/THEME` 를 첫 페인트 전에 동기로 적용: `document.documentElement.style.setProperty("--boot-bg", BGCOL)` + `U4ATheme.apply(THEME)`. body 배경은 `background: var(--boot-bg, var(--app-bg))`. 테마 적용 후 `--boot-bg` 는 제거([[boot-bg-theme-sync]] 와 동일). **iframe 본문이 있는 2단 구조면 iframe 도** 동일하게 — 부모가 `iframe.src` 에 `?BGCOL=&THEME=` 를 붙이고 iframe `<head>` 에도 같은 boot 스크립트를 둔다(안 하면 iframe 만 흰 번쩍).
- **등장 효과 = 본문 CSS opacity transition.** 네이티브 창 페이드 대신 본문 컨테이너를 `opacity:0; transition:opacity .3s` 로 두고, 준비 완료(로딩/데이터 도착) 시 클래스 토글로 `opacity:1`. GPU 합성이라 가볍고 흰 플래시 없음. `prefers-reduced-motion` 시 transition off. (예: editorPopup `#editorContent.u4aEdShown`, OTR `#otrContent.u4aOtrShown`.)
- 적용: `editorPopup`, `errPageEditorPopup`, `errMsgPopup`, `optionPopup`, `OTRF4HelpPopup`. (창 레벨 공통 UX 전체 체크리스트는 06.팝업 + ServerList 레퍼런스.)

## 3. 트리(Tree) — 공통 베이스 `U4AUI.createTree`

### 3.1 단일 출처 (필수)
- 모든 트리(서버리스트 / WS20 디자인트리 / WS30 USP / 향후 추가)는 **공통 베이스 트리 하나**를 소비한다.
  - **로직**: `theme/u4a-ui.js` 의 `U4AUI.createTree(cfg)` — 재귀 마크업·토글·들여쓰기·펼침/접힘 API.
  - **스타일**: `theme/shell.css` 의 `.u4a-tree__*` 공통 컴포넌트(토큰만 소비, 하드코딩 0).
- **금지**: 화면마다 재귀 렌더러(`_renderNode`/`_renderTreeNode` 류)를 새로 작성하는 것 = 소스 이원화. 과거 3벌 복제로 ARIA·들여쓰기·토글이 화면마다 어긋났던 문제를 막기 위함.

### 3.2 코어 UX 규칙 (베이스가 강제)
- **ARIA role 미부착.** `role="tree/treeitem/group"` 을 붙이지 않는다. 상태는 `aria-expanded`(셰브론 회전)·`aria-selected`(선택 강조)만 사용한다.
- **토글 = WS20식.** 자식은 항상 렌더하고, 펼침=`<ul>.hidden=false`(제자리), 접기=이 노드+**자손까지 재귀 접힘** 후 서브트리 재빌드(다시 펴면 자손은 접힌 상태). USP의 "매 토글 전체 재렌더"는 폐기.
- **Expand/Collapse 툴바 버튼 = "선택한 노드" 기준**(원본 `getSelectedIndex` / USP `fnUspTreeExpand·CollapseSelected`): **Expand = 선택 노드의 서브트리 전체 펼침**(루트 선택이면 트리 전체=구 `expandToLevel(99)`, 공통 `expandSubtree(node)`), **Collapse = 선택 노드만 접음(단일, `setExpanded(node,false)`)**. **선택 없으면 no-op**(전체 펼침/접힘 아님). 로드 직후 첫 루트가 선택돼 있어 기본은 전체. 버튼 툴팁은 메시지키(`C27`/`C28`, `/U4A/CL_WS_COMMON`) — 하드코딩 금지. (자체 렌더 트리도 이 동작을 복제: AppF4 가 처음에 전체펼침/접힘으로 잘못 만들었다가 2026-06-24 선택기준으로 수정.)
- **들여쓰기 표준 1종.** 행에 `--u4a-tree-depth`(=레벨)만 부여하고 padding 은 shell.css 가 계산: `calc(0.375rem + 깊이 × 1rem)`. 좁은 패널은 컨테이너 쿼리가 step(1rem→0.625rem)을 줄인다. **화면별 인라인 들여쓰기 공식 금지.**
- **DOM 계약(외부 코드가 의존하므로 고정).** `<ul class="u4a-tree">` > `<li>` > `:scope > .u4a-tree__row` + `:scope > ul`(자식). 펼침/선택 복원 로직이 이 구조와 `row._nodeData` 에 의존한다. 행 직속 자식 순서는 `[toggle][slotLead][icon][label][slotTrailing]`(별도 RowLeft 래퍼 없음 — 화면이 만든 `*RowLeft` 류는 死코드).
- ⚠️ **행 밀도(높이·간격·텍스트 크기)는 shell.css `.u4a-tree*` 단일 출처**다(2026-06-23 통일). 화면별로 다시 정의하지 않는다:
  - 행 높이 = `.u4a-tree__row { min-height: var(--u4a-tree-row-h, 2.5rem) }`(=40px, 내용 많으면 자동 증가)
  - 토글↔아이콘↔라벨 간격 = `.u4a-tree__row { gap: 0.375rem }`
  - 토글(화살표) 박스 = `.u4a-tree__toggle { 1.25rem 정사각 }`(=20px)
  - 노드 텍스트 = `.u4a-tree { font-size: 0.8125rem }`(=13px), `font-family` 는 상속(전 트리 동일)
  - **금지**: 화면 CSS 에서 `height`/`--u4a-tree-row-h`/`gap`/`font-size`/아이콘 `margin` 으로 위 값을 덮는 것. (과거 ServerList .4rem·.85rem / WS20 .25rem·13px·토글28px / USP gap+아이콘 .25rem·row-h 2.5rem / App-F4 .125rem·14px·step1.25 로 4개가 제각각 → 사용자 지적으로 수렴.)
  - **App-F4(트리테이블) 예외**: 다열 테이블이라 **행 높이만 `.u4a-table`(compact) 표준**을 따르고, 토글 간격·들여쓰기 step·텍스트 크기는 위 공통값에 맞춘다.

### 3.3 확장 컨셉 — base 위에 화면별 추가기능
- **베이스는 코어 UX 만 공통.** 화면별 추가기능(체크박스, aggregation 배지, 우측 +/삭제 액션 영역, description 컬럼 등)은 베이스가 제공하는 **슬롯/후크로 주입**한다:
  - `slotLead(node,ctx)` — 토글↔아이콘 사이(예: 체크박스)
  - `slotTrailing(node,ctx)` — 라벨 뒤 우측(예: 배지/설명/액션). 반환 시 행이 `[data-u4a-tree-split]`(우측정렬)되어 shell.css 가 `space-between` 처리.
  - `rowHook(row,node,ctx)` — 행 div 직후(줄무늬/커스텀 클래스/`data-*`/`row._nodeData` stash 등)
- **확장 스타일은 현재 각 화면 CSS 가 책임진다**(예: WS20 `ws20.css`, USP `usp.css`). 새 확장 UX가 전 화면 공통 가치가 있다고 판단되면, **그때 base 옵션 + shell.css 공통 클래스로 "승격"** 해 모든 트리가 옵션만 켜서 쓰도록 한다. (승격 여부는 작업 시 결정)
- ⚠️ **선택/상태 "좌측 컬러바"는 반드시 `box-shadow: inset <Npx> 0 0 0 <color>`(좌측 가장자리 flush)로만** 만든다 — `border-left` 로 만들지 말 것. 공통 선택바(shell.css `.u4a-tree__row[aria-selected="true"]` = `inset 0.1875rem(3px) var(--accent)`)가 있는데 행에 `border-left` 를 두면 **그 두께만큼 box-shadow 가 안쪽으로 밀려** 화면마다 선택바 위치·두께가 달라 보이고(레이아웃 시프트도 발생) 사용자가 "선택 인디케이터가 다르다"고 본다(2026-06-22 WS20 트리 `border-left:2px` 사례). WS20 상태바(`hl-Indication02/04/08`)·옵션 팝업 메뉴(`.active`)도 전부 **flush box-shadow inset 3px** 로 통일. 상태바가 선택바를 덮어야 하면 색만 바꾸고 specificity 만 보강(`.u4a-tree__row.<화면클래스>.hl-*`).

### 3.4 API 요약
- `const oCtrl = U4AUI.createTree(cfg)` — cfg 콜백:
  - 데이터: `roots()`, `children(node)`, `key(node)`(고유키), `hasChildren(node)?`
  - 표현: `label(node)`, `icon(node)?`(HTML; 빈값이면 생략), `tip(node)?`(말줄임 툴팁)
  - 슬롯: `slotLead`, `slotTrailing` / 후크: `rowHook`
  - 동작: `onSelect(node,row,ctx)`, `onToggle(node,bExpanded,row)`(펼침 영속화), `initialExpanded(node,level)?`(기본 level<1), `isExpanded(node,level)?`(외부 펼침맵 위임 — §3.4.1), `virtual?`(대용량 가상스크롤 — §3.4.1)
- 컨트롤러: `{ el, render(), expandAll(), collapseAll(), expandToLevel(n), expandSubtree(node), setExpanded(node,b), setSelected(node)/selectByKey(key), scrollToKey(key), findRow(key) }`
- 펼침 상태는 base 내부에 키 기준으로 유지되며 `onToggle` 으로 화면측 영속화(레지스트리/맵)와 동기화한다. **선택 강조는 화면이 소유(onSelect)** 하거나 base(`setSelected`)가 소유하거나 — 화면당 하나만 선택한다(이중 처리 금지).

### 3.4.1 가상스크롤(`virtual`) · 외부 펼침맵 · 다열 정렬 함정 (필수)
대용량 트리(WS30 USP / WS20 디자인트리 / MIME 등)는 `cfg.virtual:true`(보이는 행만 DOM = flat+windowed)로 만든다. 이때 아래 3가지를 어기면 화면이 깨진다(실측 사례: MIME 2026-06-25).

1. **외부 펼침맵(`isExpanded`)을 주면 내장 펼침 헬퍼가 무시된다.** `cfg.isExpanded` 를 제공하면 `_isExpanded` 가 **그 콜백만** 단일 출처로 읽는다 → 내장 `expandAll/collapseAll/expandToLevel/expandSubtree/setExpanded`(베이스 내부 `_expanded` 맵 갱신)는 **반영되지 않는다.** 화면이 소유한 펼침맵을 **직접 조작한 뒤 `render()`** 해야 한다. (가상 모드는 토글=`onToggle`→외부맵 갱신→재플래튼 전제라 보통 `isExpanded`/`onToggle` 을 화면이 소유한다.)

2. **클릭 선택에 `selectByKey`/`scrollToKey` 금지(스크롤 점프).** 가상 모드 `selectByKey(key)` 는 내부적으로 `scrollToKey`(컨테이너 `scrollTop` 재계산=점프)를 호출한다. **사용자가 클릭한 행은 이미 보이므로 스크롤하면 안 된다**(원본 UI5 도 클릭 시 스크롤 안 함). 따라서:
   - **클릭 선택** = `onSelect(node,row,ctx)` 가 넘겨주는 **`row` 에 직접 `aria-selected` 적용**(다른 행 해제). `selectByKey` 호출 ✗.
   - **선택 보존** = 선택키를 화면 상태(또는 노드 플래그, WS30 `ISSEL`)에 두고 **`rowHook` 에서 `aria-selected` 재적용** → 지연로드·펼치기/접기로 가상 재렌더돼도 스크롤 없이 선택이 유지된다.
   - `selectByKey`/`scrollToKey` 는 **화면 밖 노드를 프로그램으로 보여줄 때만**(최초 진입 reveal=구 `setFirstVisibleRow`, 검색 이동 등). 레퍼런스: WS30 `fnUspTreeMarkSelected(node,bScroll)` 주석.

3. **다열(설명 컬럼) 정렬 — `slotTrailing` 만으론 흩뿌려진다.** `slotTrailing` 반환 시 행이 `[data-u4a-tree-split]`(=`space-between`)이 되는데, 라벨이 자연폭이라 토글·아이콘·라벨·설명이 가로로 흩어진다. 컬럼처럼 정렬하려면 화면 CSS 가 전부 덧대야 한다(WS30 `usp.css` / MIME 이식이 레퍼런스):
   - `.u4a-tree__label { flex: 1 1 0; min-width:0; overflow:hidden; text-overflow:ellipsis }` — 라벨이 남는 폭을 채움.
   - `.u4a-tree__row[data-u4a-tree-split] { justify-content: flex-start }` — 공통 `space-between` 무력화 + 설명은 **고정폭 우측 컬럼**(`flex:0 0 <desc-w>` + 좌측 구분선).
   - **들여쓰기를 행 `padding-left` → 토글 `margin-left`(=`calc(depth × step)`)로 이동**, 행은 `padding-left` 고정. 설명 컬럼 폭이 flex-basis(%)라 **행 content-box 폭이 깊이와 무관하게 일정**해야 컬럼/구분선이 헤더와 정렬된다.
   - sticky 컬럼 헤더는 **행과 동일한 `padding-left`** 를 주고(폭 컨텍스트 일치), 설명폭은 단일 변수(`--*-desc-w`)로 헤더·행 공유.

## 4. 스플릿바(Splitter) — 공통 `.u4a-splitter*` + 서버리스트 기준 그립

좌우/3분할 패널을 나누는 **분할 리사이저(스플릿바)** 는 화면마다 새로 만들지 않고, 공통 골격
(`shell.css` 의 `.u4a-splitter*`)과 공통 드래그 동작을 소비하며 **모양(그립)은 서버리스트 기준으로
통일**한다. 트리(§3)와 동일한 단일화 원칙이다.

### 4.1 DOM 계약 (필수)
- 컨테이너 `<div class="u4a-splitter">` 안에 **패널과 바를 번갈아** 배치한다.
  - 2분할: `pane · bar · pane`
  - 3분할(마스터-디테일): `pane · bar · pane · bar · pane`
- 패널 `<div class="u4a-splitter__pane">`, 바 `<div class="u4a-splitter__bar" role="separator">`.
- 컨테이너는 `display:flex; width/height:100%; overflow:hidden`(패널 폭 합이 넘쳐도 페이지로 전파하거나
  세로 공간을 침범하지 않게 클립) — `shell.css` 가 소유한다.
- 패널 폭은 **인라인 `flex`** 로 표현한다: 초기엔 비율(예: 좌측 `0 0 30%`, 가운데 축소형 `0 1 34%`),
  **마지막 패널은 `1 1 auto`**(잔여 채움). 드래그하면 고정 패널이 `0 0 <px>px` 로 바뀐다.
- **초기 고정 px 폭 금지**(반응형, [12번](12_테마_컨버전_전략.md) 7장) — 초기 basis 는 %/유동 단위,
  px 폭은 사용자의 드래그 결과로만 생긴다.

### 4.2 시각 표준 — 서버리스트 기준 그립 (필수)
- 바 폭 11px, 표면색 배경 + 좌우 1px 경계.
- **가운데 그립 핸들**(바 `::before`): 5px×36px 알약(radius 4px), 평소 뮤트색(`--icon-muted` 계열) opacity .5.
- **hover**: 바 배경 → **불투명 `--surface` + `--hover-bg` 틴트 오버레이**(`background-color:--surface` +
  `background-image:linear-gradient(--hover-bg,--hover-bg)`), 그립 → **accent 색 + opacity 1 + 높이 50px** 로 강조.
  ⚠️ 바 hover 에 `background: var(--hover-bg)`(반투명 단색)만 주지 말 것 — 반투명이라 surface 가 사라지고
  **바 뒤 컨테이너 배경이 비치는데, 그 컨테이너가 화면마다 달라(WS20=app-bg, 옵션=투명→body 등) 같은
  토큰인데도 hover 색이 화면마다 달라 보인다.** surface 베이스+틴트로 고정해야 전 화면 동일(2026-06-22).
- 커서 `col-resize`. 색은 전부 의미 토큰만 사용(하드코딩 hex 금지). 치수(11/5/36/50px)는 그립 형태값으로 허용한다.
- ⚠️ **바 배경을 화면별로 override 할 땐 `:hover` 도 같은 특이도로 함께 override 한다(필수).** 공통 hover 는
  `.u4a-splitter__bar:hover { background: var(--hover-bg) }`(클래스 특이도 0,2,0). 화면에서 `#id .u4a-splitter__bar { background: … }`(id 특이도 1,1,0)로 **기본 배경만** 바꾸면 그게 공통 `:hover` 를 이겨 **그 화면만
  hover 색이 사라진다.** → 반드시 `#id .u4a-splitter__bar:hover { background: var(--hover-bg) }` 도 같이 준다.
  (WS20·옵션 팝업에서 이 specificity 함정으로 hover 통일이 깨졌던 사례 — 2026-06-22.)

### 4.3 동작 — 인접 재분배 드래그 + 리사이즈 재클램프 (필수)
- **드래그**: 바를 잡으면(`mousedown`) `body` 커서를 `col-resize` 로, `mousemove` 로 **바 양옆 인접 두 패널
  사이에서만** 폭을 주고받는다(나머지 패널은 불변). 각 패널의 `min-width` 이하로는 줄지 않는다.
- **마지막(유연) 패널 보호**: 오른쪽 이웃이 마지막 `1 1 auto` 패널이면 왼쪽만 px 로 두고 오른쪽은 flex 로
  잔여를 채운다 → 마지막 패널(상세 등)이 짜부되어 사라지지 않는다.
- **창 리사이즈 재클램프**: 창이 줄면 `px` 고정 패널 합 + 바 + 유연패널 최소폭이 컨테이너를 넘지 않도록
  **큰 고정 패널부터 min 까지 자동 축소**한다(`overflow:hidden` 에 잘려 마지막 패널/바가 숨는 것 방지).
  유연/% 패널은 CSS `flex`+`min-width` 로 알아서 줄므로 **px 고정분만** 처리한다. 리스너는 **1회만 바인딩**하고,
  뷰 전환으로 스플릿이 새로 그려져도 `querySelector` 로 그 시점의 스플릿을 처리한다.
- min-width 폴백 120px.
- **더블클릭 = 최초 위치 복귀(공통·전역 자동, 승격 완료)**: 바를 더블클릭하면 인접 패널이 **처음(드래그 전) 폭**으로
  돌아간다. `u4a-ui.js _installGlobalSplitterReset()` 가 모듈 로드 시 1회 자동 설치 — 모든 `.u4a-splitter__bar` 에
  **위임 1개**로 적용(화면별 배선 불필요, 다이얼로그 드래그와 동일 패턴). 최초 폭은 각 패널이 "처음 드래그되기
  직전"(document `mousedown` **capture** → 화면별 드래그보다 먼저 실행)의 인라인 `style.flex`(없으면 `""`=CSS
  기본)를 `data-u4aSplitHome` 에 **1회만** 기록(이미 있으면 안 덮음 → 두 바가 공유하는 패널(3분할 가운데)도 최초값
  유지), 더블클릭 시 바의 prev/next 형제(=인접 패널, 클래스 무관)를 그 값으로 복귀. 드래그/재클램프(아직 화면별
  복제)와 달리 **이 동작은 단일 전역 헬퍼로 이미 승격됨** → 새 분할 화면은 바에 `.u4a-splitter__bar` 만 붙이면 자동.

> **필수 엣지 케이스 — 최대화 후 restore 시 바/패널이 숨는 문제**
> 브라우저(창) **최대화 상태에서 스플릿바를 오른쪽 끝까지 드래그**하면 좌측 패널이 큰 `px` 폭으로 고정된다.
> 이 상태로 창을 **restore(축소)** 하면, 고정 px 폭 합 + 바 폭이 줄어든 컨테이너를 초과해 `overflow:hidden` 에
> 뒤쪽 패널·바가 잘려 **사라진 것처럼 보인다.** → 위 "창 리사이즈 재클램프"가 이를 막는 **필수** 처리다.
> 새 분할 화면은 반드시 이 재클램프를 바인딩한다.
> 레퍼런스(동일 패턴): ServerList `_bindSplitterResizeClamp`, WS20 `_bindWs20SplitResizeClamp`.

## 5. 다국어(i18n) / 텍스트

화면에 보이는 **모든 문구는 SQLite 메시지 클래스의 키**에서 가져온다. 문자열을 소스에 직접 적지 않는다
(하드코딩 금지). 컨버전·신규 화면 공통 규칙이며, 메시지 키 사용 원칙은 [02번](02_로그인_서버리스트.md)·
[15번](15_공통_입력UX_가이드.md)과 동일하다.

### 5.1 공통 추출 함수 (필수)
- **저수준**: `WSUTIL.getWsMsgClsTxt(LANGU, MSGCLS, MSGNR, p1, p2, p3, p4)` — SQLite 에서 언어/클래스/번호로
  텍스트를 조회하고 `&1`~`&4`(및 `&`) 플레이스홀더를 `p1`~`p4` 로 치환한다.
- **래퍼**: `fnGetMsgClsText(MSGCLS, MSGNR, p1..p4)` — `LANGU` 를 워크스페이스 언어로 해석한 뒤 위 저수준 함수를
  호출한다. 래퍼 진입점은 화면마다 다를 수 있으나(예: ServerList `MessageClassText.fnGetMsgClsText`,
  WS `oAPP.common.fnGetMsgClsText`) **모두 `WSUTIL.getWsMsgClsTxt` 로 위임**된다 — 새로 만들지 말고 그 화면 래퍼를 쓴다.
- 화면 코드는 위(저수준/래퍼) 중 하나만 쓴다. **리터럴 문자열을 DOM 에 직접 넣지 않는다.**

### 5.2 언어 기준
- 라벨 언어 = **워크스페이스(접속 사용자) 언어** — `parent.process.USERINFO.LANGU`(있으면 최우선),
  WS UI 언어는 `WSUTIL.getWsLanguAsync()`. **백엔드 `serverInfo.LANGU` 가 아니다.**
- 메시지 클래스 예: `ZMSG_WS_COMMON_001`(번호 M0271 등), `/U4A/CL_WS_COMMON`(번호 A68/A69 등).

### 5.3 금지
- 하드코딩 문자열, **영문(또는 임의) 폴백 기본값**, 완료 토스트 등 **임의 문구 생성**, **가짜/추측 키** 사용 금지.
- 기존 키가 있으면 **그대로 재사용**한다(같은 의미의 키 중복 생성 금지).

### 5.4 신규 텍스트가 필요할 때 (작업 흐름)
화면에 **기존 키가 없는 새 문구**가 필요하면:
1. 소스에는 **공통 추출 함수 호출만** 넣는다(하드코딩·자리 임시문구 금지). 그 자리에서 임의 키를 확정하지 않는다.
2. 작업 중 발생한 **신규 텍스트를 모았다가, 작업 끝에 "신규 항목만" 목록으로 제출**한다(아래 양식).
3. 사용자가 그 목록을 **번역하여 SQLite DB 에 추가/업데이트 → 앱 재실행**한다.

**신규 텍스트 제출 양식**(예시):

| # | 메시지클래스(제안) | 번호(제안) | 한국어 원문 | 플레이스홀더 | 사용 위치(파일 · UI) |
|---|---|---|---|---|---|
| 1 | /U4A/CL_WS_COMMON | (신규) | 서버를 선택하세요 | – | ServerList.js · 빈 상태 안내 |
| 2 | /U4A/CL_WS_COMMON | (신규) | &1 개 항목이 선택됨 | `&1`=개수 | (예) 다중선택 푸터 |

- 클래스/번호는 **제안**이며, 최종 확정은 사용자(번역·DB 입력) 몫이다.
- 플레이스홀더가 있으면 `&1`~ 표기와 의미를 함께 적는다.

## 6. 테이블(Table) — 공통 `.u4a-table` + 컬럼 정렬/필터 메뉴 (서버리스트 기준)

목록/그리드는 **공통 테이블 컴포넌트**를 소비한다. 화면마다 표 스타일·정렬·필터를 새로 만들지 않는다.

### 6.1 단일 출처 (필수)
- 마크업: `.u4a-table-wrap > table.u4a-table`(`shell.css`).
- **래퍼(컨테이너) 외형도 공통 단일 출처** — 화면마다 손으로 깔지 말 것(이걸 안 지켜 CSS Link 팝업 테이블이 AppF4/Insert 와 달라 보였던 사례, 2026-06-25):
  - **팝업/다이얼로그 안 테이블 = 액자형 `.u4a-table-wrap.u4a-table-wrap--boxed`**(`shell.css`: `overflow:auto` + `border:var(--line)` + `border-radius:var(--radius)` + `background:var(--surface)` + `flex:1 1 auto;min-height:0`). AppF4 F4팝업·Insert Element 팝업·CSS/JS Link 팝업 모두 **이 한 클래스**로 동일 외형. 화면 CSS 에서 `border`/`radius`/`background` 직접 칠하지 말 것(`--divider`·하드코딩 radius 금지 → 색·굵기 어긋남).
  - **풀페이지 보더리스 테이블(ServerList)** 은 `--boxed` 를 안 붙인다(테두리 없음, 세로 스크롤만). 즉 액자 여부만 모디파이어로 가른다.
- ⚠️ **`.u4a-table--compact` 는 WS 앱에서 쓰지 않는다** — 혼자 행이 좁아져 다른 표와 달라 보인다(insert·errMsg 가 붙였다 제거). 행높이는 공통 기본(`--row-h` 2.5rem) 단일값. 새 테이블에 `--compact`/`height`/`padding` 으로 행높이 건드리지 말 것.
- 헤더 sticky, zebra=행 `data-odd="true"`, hover, **선택=행 `aria-selected="true"`(좌측 accent 바)**, 빈 결과=`tr.u4a-table__nodata`, 포커스 링.
- 카드(타일) 뷰 `.u4a-table-wrap[data-view="card"]`(컨테이너쿼리 미지원 → 화면 JS 가 ResizeObserver 로 `data-view` 토글)는 **옵트인 기능**이다.
  - ⚠️ **카드 반응형은 사용자/원본의 별도 언급 없이는 적용하지 않는다.** 기본값은 `data-view="table"`(가로 스크롤로 대응). "공통 표준에 있으니까" 자동으로 끼워넣지 말 것 — 특히 **인라인 편집 그리드**(입력칸·스위치·체크박스 행)는 카드 뷰가 편집 UX 를 해친다. 필요하면 그때 명시 요청으로만 켠다. (근거: 요청 밖 UX 추가 금지.)
- `.u4a-table*` 는 전부 `shell.css` 공통. 화면별 확장(컬럼 폭·정렬메뉴 헤더 등)만 `.u4a-c-*`/스코프로 얹는다.

### 6.2 컬럼 헤더 정렬/필터 메뉴 — 서버리스트 기준 (필수)
원본 UI5 `sap.ui.table` 의 `sortProperty`/`filterProperty`(헤더 클릭 시 정렬·필터 메뉴)는 **서버리스트 `fnOpenColumnMenu` 패턴으로 통일**한다.
- 정렬·필터 가능한 `th` 에 `.u4a-th--menu`(클릭) + 내부 `.u4a-th__inner`(라벨 `.u4a-th__label` + 표시자 `.u4a-th__ind`).
- 클릭 → 공통 `.u4a-menu.u4a-colmenu`. **구성·순서(서버리스트 동일)**: ① 필터 input(`.u4a-colmenu__filter > .u4a-input`) ② 구분선 ③ 오름차순 정렬 ④ 내림차순 정렬 ⑤ 구분선 ⑥ 필터 초기화(이 컬럼).
- **필터**: 표시 텍스트 기준 contains(대소문자 무시), 여러 컬럼 **AND**, Enter/blur 시 적용. **정렬**: 단일 컬럼 `localeCompare(…,{numeric:true})`, 같은 방향 재클릭 시 해제. 활성 정렬 항목=`.u4a-menu__item[data-active="true"]`.
- **표시자**(`.u4a-th__ind`): 정렬 걸린 컬럼=화살표(arrow-up/down), 필터 걸린 컬럼=깔때기(funnel). 걸린 컬럼만 노출.
- **원본 배열(검색결과 등)은 보존**하고 화면에 그릴 **뷰만 필터→정렬로 파생**(비파괴). 헤더는 매 렌더 재구성(표시자 갱신).
- 메시지 키: 오름/내림 정렬=`ZMSG_WS_COMMON_001` 810/811, 필터 값(placeholder)=`/U4A/CL_WS_COMMON` A68, 필터 초기화=A69.
- ⚠️ `.u4a-th__*`·`.u4a-colmenu*` CSS 는 현재 **서버리스트 전용(`serverlist.css`)** — 다른 창은 스코프로 복제(공통 `.u4a-menu`/`.u4a-menu__item` 은 `shell.css`). **향후 `shell.css` 승격 권장.** showModal 다이얼로그 안 테이블이면 메뉴를 **다이얼로그(top-layer) 안에 append**(밖이면 모달에 가림) + `getBoundingClientRect`+`position:fixed` 로 헤더 아래 배치.
- ⚠️ **메뉴는 컨트롤러로 일반화해 공유**: `_openColMenu(c, th, ctl)` 가 상태를 직접 만지지 말고 컨트롤러(`getFilter/setFilter/getSort/setSort/rerender`)를 받게 한다 → 한 화면에 표가 여럿(예: F4 탭1 결과표 + 탭2 트리표)이면 컨트롤러만 따로 주고 메뉴·표시자·정렬·필터 로직은 1벌 공유. 상태 변수만 표별로 분리.
- ⚠️ **sticky 헤더 hover 배경의 반투명 함정**: `th:hover{background:var(--hover-bg)}` 만 주면 `--hover-bg`(rgba ~.08, 92% 투명)가 불투명 `--surface` 를 덮어 **헤더 뒤로 스크롤되는 데이터가 비쳐 헤더 글자가 "겹쳐" 보인다**. 반드시 `background:linear-gradient(var(--hover-bg),var(--hover-bg)),var(--surface)` 로 불투명 surface 위에 틴트를 얹는다(프로즌 셀·스플릿바 hover 와 동일 레이어 문법, §4.2).
- **정렬·필터 전체 해제 버튼(권장)**: 정렬 가능한 컬럼이 많으면 메뉴로 일일이 끄기 번거로우므로 **결과 테이블 위 툴바**에 한 번에 초기화하는 아이콘 버튼(`filter-circle-xmark`)을 둔다. 걸린 게 없으면 **비활성**(`:disabled`), 툴팁은 per-column 과 같은 의미라 **A69** 재사용. **범위 = 클라이언트 뷰 파생(헤더 메뉴: 정렬+컬럼필터)만 리셋**, 서버 쿼리(검색 폼 입력)는 유지(F4 탭1 기준). 단 그 화면의 검색박스가 클라이언트 필터면(F4 탭2 트리) 같이 초기화한다. F4 탭1·탭2 모두 적용. ⚠️ 툴바 아이콘 버튼은 화면별로 크기 줄이지 말 것 — 공통 `.u4a-btn-icon` 표준 2rem(행 안 액션 버튼만 컴팩트 허용, override 는 `.u4a-table` 스코프로 가둔다).

### 6.2.1 트리 테이블(TreeTable)의 컬럼 정렬/필터 — 트리 의미로 변형
원본 UI5 `TreeTable` 컬럼도 `sortProperty`/`filterProperty` 가 없었다(F4 두 탭 모두). 위 6.2 헤더메뉴를 트리에 이식하되 **트리 의미로 변형**한다(F4 탭2 기준).
- **정렬 = 형제 노드끼리만**(각 부모의 자식 배열을 정렬키로 재정렬, 계층 구조·깊이는 유지 — UI5 TreeTable sorter 동일). **비파괴**(원본 배열 보존, 평탄화 단계에서 정렬뷰 생성). 최상위 배열도 정렬.
- **필터 = 경로 유지 contains**: 노드 자체 매칭 = (상단 검색박스 매칭) **AND** (컬럼필터 매칭). 최상위 컨테이너 노드는 항상 통과. 자손에 매칭이 있으면 **조상 경로를 표시**(트리 끊김 방지). 매칭이 0이면 root 까지 빠져 공통 no-data.
- 컬럼필터 변경/탭 재진입(재조회) 시에도 **매칭 경로 자동 펼침**(검색박스와 동일 규칙) — 필터 활성이면 펼침 모습 유지.
- **전체 해제 버튼**: 컬럼 헤더가 여러 개라 정렬·필터를 일일이 메뉴로 끄기 번거로우므로, 툴바에 **정렬·컬럼필터·검색박스를 한 번에 초기화하는 버튼**(아이콘 `filter-circle-xmark`)을 둔다. 걸린 게 하나도 없으면 **비활성**(`:disabled`). 툴팁은 per-column 초기화와 같은 의미라 메시지키 **A69**(필터 초기화) 재사용. 검색박스도 비울 땐 `input` 이벤트를 디스패치해 clear(X) 글리프까지 동기화.

### 6.3 적용 현황
- 레퍼런스: 서버리스트(`ServerList.js` `fnOpenColumnMenu`/`_buildServerView`).
- 소비: errMsgPopup, App Search Help F4 팝업(탭1 결과 테이블 + **탭2 트리 테이블** = 6.2.1 변형).

## 7. 플로팅 푸터 메시지(Footer) — 공통 `U4AUI.footer*` + `.u4a-footer`

화면 하단에 잠깐 떠오르는 상태/결과 메시지(구 sap.m floatingFooter). **단일 출처 = `U4AUI.footerMarkup/footerShow/footerHide`(u4a-ui.js) + shell.css `.u4a-footer*`.** 화면마다 마크업/show·hide 를 복제하지 않는다.

### 7.1 단일 출처 (필수)
- 마크업: `U4AUI.footerMarkup(sId)` 가 반환하는 문자열을 화면 마크업에 삽입. 구조 `.u4a-footer[data-show][data-type]` > `.u4a-footer__icon` + `.u4a-footer__text` + `.u4a-footer__close`(X, `data-u4a-footer-close`).
- 표시/숨김: `U4AUI.footerShow(elOrId, type, msg, ms)` / `U4AUI.footerHide(elOrId)`. `type` = E/S/W/I(아이콘·색 자동), `ms` 생략=10000, **`ms=0`=자동숨김 없음**(타이머를 호출측 래퍼가 소유할 때).
- **닫기(X)는 전역 위임 1개**(`document` click → `[data-u4a-footer-close]`) — 화면별 `addEventListener` 금지.

### 7.2 코어 UX 규칙 (베이스가 강제)
- 색은 의미 토큰만(`--state-error/success/warning/info`, `--surface`/`--line`/`--popover-shadow`). 하드코딩 금지.
- 노출 시 1회 슬라이드업 `animation`(종료 `transform:none` → 잔여 컴포지팅 레이어 없음 = 프레임리스 타이틀바 드래그영역 무영향). `prefers-reduced-motion` 시 애니메이션 off.
- 문구는 메시지 키만(임의 생성 금지, §5).

### 7.3 적용 현황
- 소비: WS10(통합 `js/ws10_html.js` + 독립 프리뷰 `WS10/WS10.js`), WS20(`ws_html5_ws20.js`), WS30 USP(`js/usp/ws_html5_usp.js`). 각 화면의 `oAPP.{ws10html|ws20html|usphtml}.showFooter/hideFooter` 는 이름 유지하고 내부만 `U4AUI` 위임(셸 `fnShowFloatingFooterMsg` 라우팅 무수정).
- 이관 전: CSS만 `.u4a-ws10__footer`(ws10.css) 공유 + 로직 복제 → WS30 만 닫기(X) 누락이었음(2026-06-23 통일, 구 CSS 제거).

> 참고: 컨텍스트 메뉴 그룹 구분선은 공통 `.u4a-menu__sep`(shell.css). 트리 서브트리 펼침은 `U4AUI.createTree().expandSubtree(node)`.

## 8. 패널(Panel) — 공통 `U4AUI.createPanel` + `.u4a-panel*`

### 8.1 단일 출처 (필수)
- 접이식 조회/속성 영역(원본 `sap.m.Panel` expandable)은 **공통 패널 하나**를 소비한다.
  - **로직**: `theme/u4a-ui.js` 의 `U4AUI.createPanel(cfg)` — 헤더/토글/접힘 상태(`data-collapsed`) 관리.
  - **스타일**: `theme/shell.css` 의 `.u4a-panel*`(토큰만, 하드코딩 0).
- **금지**: 화면마다 패널 골격(섹션+헤더버튼+바디+접힘 토글)을 새로 짜는 것 = 소스 이원화.

### 8.2 구조 / API
- DOM: `<section.u4a-panel[data-collapsed]>` > `.u4a-panel__head`(= `.u4a-panel__toggle`(twisty+제목) + `.u4a-panel__actions` 슬롯) + `.u4a-panel__body`. ★ 헤더에 **검색 버튼 등 액션**을 넣을 수 있게 head 는 `div`, 접기 토글만 `button`(버튼 중첩 회피). twisty 회전은 CSS(`:not([data-collapsed="X"])` → 90°), innerHTML 스왑 안 함.
- 헤더 높이 `min-height:2.75rem`(액션 버튼이 들어가도 균형). **헤더 배경 = `--hover-bg` 상시 틴트**(`.u4a-panel__head`)로 본문과 시각 구분(옛 hover 색을 기본으로 올림), **hover 시 `--active-bg` 로 살짝만 더 강조**(헤더 전체가 한 덩어리 — 검색 버튼도 같은 밴드). 토글만 hover 시키면 좌측만 칠해져 액션 버튼이 분리돼 보였음. 토글은 `align-self:stretch` 로 헤더 높이만큼 클릭(접기) 영역 확보.
- `createPanel({title, collapsed, onToggle})` → `{ el, head, body, actions, toggle, setCollapsed(b), isCollapsed() }`. 화면은 `body`에 내용, `actions`에 헤더 버튼을 append.
- 외부에서 프로그램적 접기: `el` 에 `data-collapsed="X"` 세팅(공통 CSS 가 바디 숨김+twisty 회전 처리) + 필요 시 `.u4a-panel__toggle` aria-expanded 동기. (USP 에디터 전체화면 토글이 이 방식.)

### 8.3 적용 현황
- 소비: **WS30 USP Properties 패널**(`js/usp/ws_html5_usp.js _buildUspPanel`, 화면은 여백/flex만 `.u4aWs30Panel` 로 확장), **AppF4 F4팝업 조회조건 패널**(`fnAppF4PopupOpen.js`, 헤더 actions=검색 버튼, 바디=조회 폼). (2026-06-23 USP 전용 `u4aWs30Panel*` → 공통 이관, F4 신규 소비.)

## 9. 원칙 요약

- **모든 화면 = Bootstrap 베이스 + tokens.css 테마.**
- **모달/메시지 박스 = 헤더 48px · 푸터 48px · 드래그 · 리사이즈 · 창 리사이즈 재클램프**(드래그/리사이즈된 팝업이 창 restore 시 잘리지 않게 전역 클램프, §2.2).
- **트리 = 공통 베이스 `U4AUI.createTree` 소비** (role 미부착 · 토글 WS20식 · 들여쓰기 `--u4a-tree-depth` 표준 · 추가기능은 slot/hook 확장).
- **스플릿바 = 공통 `.u4a-splitter*` + 인접 재분배 드래그 · 리사이즈 재클램프 소비** (모양은 서버리스트 기준 그립으로 통일, 초기 고정 px 폭 금지).
- **테이블 = 공통 `.u4a-table` 소비 + 컬럼 정렬/필터 메뉴는 서버리스트 기준** (`.u4a-th--menu`→`.u4a-colmenu`: 필터 input · 오름/내림 정렬 · 초기화 / 필터 contains·AND, 정렬 단일·numeric, 원본 보존+뷰 파생).
- **플로팅 푸터 = 공통 `U4AUI.footer*` + `.u4a-footer` 소비** (닫기X 전역위임 · 자동숨김 · 타입 아이콘/색 토큰, 화면별 복제 금지).
- **접이식 패널 = 공통 `U4AUI.createPanel` + `.u4a-panel*` 소비** (원본 sap.m.Panel, 헤더 토글+액션 슬롯 / 바디, 화면별 패널 골격 새로 짜지 말 것 — §8).
- **다국어 = 모든 문구를 SQLite 메시지 키 + 공통 추출 함수(`WSUTIL.getWsMsgClsTxt`/`fnGetMsgClsText`)로** (하드코딩·임의키 금지, 신규 텍스트는 목록 제출 → 사용자 번역·DB 입력).
- 공통 자산(Bootstrap, tokens.css, shell.css, u4a-ui.js)을 **소비**하고 화면마다 새로 만들지 않는다.
