# 16. 공통 화면 UX 표준

> 모든 화면에 **공통으로 적용**되는 UX 표준(전역 규칙)이다. 특정 기능 문서(02~08)나
> 입력 가이드(15)보다 상위의 "모든 화면 공통" 기준이며, 화면을 새로 만들거나
> U4A WS → HTML5 로 변환할 때 항상 함께 적용한다.
> 관련: [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md)(테마 토큰), [15_공통_입력UX_가이드.md](15_공통_입력UX_가이드.md)(입력칸), [06_팝업.md](06_팝업.md)(팝업 카탈로그), [02_로그인_서버리스트.md](02_로그인_서버리스트.md)(트리 기준)

## 0. TL;DR

1. 모든 화면은 **Bootstrap** 을 기준(베이스) 프레임워크로 사용한다.
2. 색·간격 등 **테마는 `tokens.css` 의미 토큰**으로 입힌다 (Bootstrap CSS 변수에 매핑). 하드코딩 금지.
3. 모든 메시지 박스·모달 팝업의 **헤더/푸터 높이는 각각 48px**.
4. 모달 팝업은 **드래그(이동) + 리사이즈(크기 조절)** 가능해야 한다.
5. 트리는 **공통 베이스 트리 `U4AUI.createTree`(`u4a-ui.js`)** 를 소비한다. 화면마다 재귀 렌더러를 새로 만들지 않는다(소스 단일화).
6. 스플릿바(분할 리사이저)는 **공통 골격 `.u4a-splitter*`(`shell.css`) + 인접 재분배 드래그**를 소비하고, 모양은 **서버리스트 기준(가운데 그립 핸들)** 으로 통일한다.
7. 화면 문구는 **전부 SQLite 메시지 클래스 키 + 공통 추출 함수**로 표시한다(하드코딩 금지). 기존 키가 없는 신규 텍스트는 **임의 생성·임의 키 금지**, 모았다가 **신규 항목만 목록으로 제출** → 사용자가 번역·DB 입력·재실행.
8. 목록/그리드는 **공통 `.u4a-table`** 을 소비하고, **컬럼 헤더 정렬/필터 메뉴는 서버리스트 기준**(필터 input → 오름/내림 정렬 → 필터 초기화)으로 통일한다.

## 1. 기준 프레임워크 — Bootstrap

- **모든 화면은 Bootstrap 을 베이스로 사용한다.** 레이아웃·그리드·버튼·폼 등 표준 컴포넌트는
  Bootstrap 컴포넌트/클래스를 우선 소비하고, 화면마다 새로 만들지 않는다.
- **테마는 토큰으로 입힌다.** `theme/tokens.css` 의 의미 토큰(색/간격)을 **Bootstrap CSS 변수
  (`--bs-*`)에 매핑**하여 적용한다. 색을 직접 hex 로 박지 않는다(드리프트 금지). 테마 단일 출처는
  [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md).
- **공통 컴포넌트는 Bootstrap "위에" 얹는다.** 입력칸의 `.u4a-field`(clear/스피너 제거 등,
  [15번](15_공통_입력UX_가이드.md))과 `shell.css` 의 셸 스타일은 Bootstrap 베이스 위에서 동작하는
  프로젝트 공통 자산으로 유지한다. Bootstrap 과 겹치면 프로젝트 공통 컴포넌트 규칙을 우선한다.

> 주의: `.analy` 다른 문서에서 말하는 "bootstrap"은 대개 제거 대상인 **UI5 로더(`sap-ui-bootstrap`)**
> 를 뜻한다. 본 문서의 "Bootstrap"은 **CSS 프레임워크 Bootstrap** 을 가리키며 서로 다른 개념이다.

## 2. 메시지 박스 / 모달 팝업

### 2.1 헤더·푸터 높이
- 모든 메시지 박스와 모달 팝업의 **헤더(header) 높이는 48px**.
- 모든 메시지 박스와 모달 팝업의 **푸터(footer) 높이는 48px**.
- 본문(body)은 헤더/푸터를 제외한 나머지 영역을 차지한다.

### 2.2 드래그 + 리사이즈
- 모달 팝업은 **헤더를 잡고 드래그하여 이동**할 수 있어야 한다.
- 모달 팝업은 **가장자리/모서리로 리사이즈(크기 조절)** 할 수 있어야 한다.
- 팝업 카탈로그와 개별 교체 방식은 [06_팝업.md](06_팝업.md)를 따른다.

## 3. 트리(Tree) — 공통 베이스 `U4AUI.createTree`

### 3.1 단일 출처 (필수)
- 모든 트리(서버리스트 / WS20 디자인트리 / WS30 USP / 향후 추가)는 **공통 베이스 트리 하나**를 소비한다.
  - **로직**: `theme/u4a-ui.js` 의 `U4AUI.createTree(cfg)` — 재귀 마크업·토글·들여쓰기·펼침/접힘 API.
  - **스타일**: `theme/shell.css` 의 `.u4a-tree__*` 공통 컴포넌트(토큰만 소비, 하드코딩 0).
- **금지**: 화면마다 재귀 렌더러(`_renderNode`/`_renderTreeNode` 류)를 새로 작성하는 것 = 소스 이원화. 과거 3벌 복제로 ARIA·들여쓰기·토글이 화면마다 어긋났던 문제를 막기 위함.

### 3.2 코어 UX 규칙 (베이스가 강제)
- **ARIA role 미부착.** `role="tree/treeitem/group"` 을 붙이지 않는다. 상태는 `aria-expanded`(셰브론 회전)·`aria-selected`(선택 강조)만 사용한다.
- **토글 = WS20식.** 자식은 항상 렌더하고, 펼침=`<ul>.hidden=false`(제자리), 접기=이 노드+**자손까지 재귀 접힘** 후 서브트리 재빌드(다시 펴면 자손은 접힌 상태). USP의 "매 토글 전체 재렌더"는 폐기.
- **들여쓰기 표준 1종.** 행에 `--u4a-tree-depth`(=레벨)만 부여하고 padding 은 shell.css 가 계산: `calc(0.375rem + 깊이 × 1rem)`. 좁은 패널은 컨테이너 쿼리가 step(1rem→0.625rem)을 줄인다. **화면별 인라인 들여쓰기 공식 금지.**
- **DOM 계약(외부 코드가 의존하므로 고정).** `<ul class="u4a-tree">` > `<li>` > `:scope > .u4a-tree__row` + `:scope > ul`(자식). 펼침/선택 복원 로직이 이 구조와 `row._nodeData` 에 의존한다.

### 3.3 확장 컨셉 — base 위에 화면별 추가기능
- **베이스는 코어 UX 만 공통.** 화면별 추가기능(체크박스, aggregation 배지, 우측 +/삭제 액션 영역, description 컬럼 등)은 베이스가 제공하는 **슬롯/후크로 주입**한다:
  - `slotLead(node,ctx)` — 토글↔아이콘 사이(예: 체크박스)
  - `slotTrailing(node,ctx)` — 라벨 뒤 우측(예: 배지/설명/액션). 반환 시 행이 `[data-u4a-tree-split]`(우측정렬)되어 shell.css 가 `space-between` 처리.
  - `rowHook(row,node,ctx)` — 행 div 직후(줄무늬/커스텀 클래스/`data-*`/`row._nodeData` stash 등)
- **확장 스타일은 현재 각 화면 CSS 가 책임진다**(예: WS20 `ws20.css`, USP `usp.css`). 새 확장 UX가 전 화면 공통 가치가 있다고 판단되면, **그때 base 옵션 + shell.css 공통 클래스로 "승격"** 해 모든 트리가 옵션만 켜서 쓰도록 한다. (승격 여부는 작업 시 결정)
- ⚠️ **선택/상태 "좌측 컬러바"는 반드시 `box-shadow: inset <Npx> 0 0 0 <color>`(좌측 가장자리 flush)로만** 만든다 — `border-left` 로 만들지 말 것. 공통 선택바(shell.css `.u4a-tree__row[aria-selected="true"]` = `inset 0.1875rem(3px) var(--accent)`)가 있는데 행에 `border-left` 를 두면 **그 두께만큼 box-shadow 가 안쪽으로 밀려** 화면마다 선택바 위치·두께가 달라 보이고(레이아웃 시프트도 발생) 사용자가 "선택 인디케이터가 다르다"고 본다(2026-06-22 WS20 트리 `border-left:2px` 사례). WS20 상태바(`hl-Indication02/04/08`)·옵션 팝업 메뉴(`.active`)도 전부 **flush box-shadow inset 3px** 로 통일. 상태바가 선택바를 덮어야 하면 색만 바꾸고 specificity 만 보강(`.u4a-tree__row.<화면클래스>.hl-*`).

### 3.4 API 요약
- `const oCtrl = U4AUI.createTree(cfg)` — cfg 콜백:
  - 데이터: `roots()`, `children(node)`, `key(node)`(고유키), `hasChildren(node)?`
  - 표현: `label(node)`, `icon(node)?`(HTML; 빈값이면 생략), `tip(node)?`(말줄임 툴팁)
  - 슬롯: `slotLead`, `slotTrailing` / 후크: `rowHook`
  - 동작: `onSelect(node,row,ctx)`, `onToggle(node,bExpanded,row)`(펼침 영속화), `initialExpanded(node,level)?`(기본 level<1)
- 컨트롤러: `{ el, render(), expandAll(), collapseAll(), expandToLevel(n), setExpanded(node,b), setSelected(node)/selectByKey(key), findRow(key) }`
- 펼침 상태는 base 내부에 키 기준으로 유지되며 `onToggle` 으로 화면측 영속화(레지스트리/맵)와 동기화한다. **선택 강조는 화면이 소유(onSelect)** 하거나 base(`setSelected`)가 소유하거나 — 화면당 하나만 선택한다(이중 처리 금지).

### 3.5 적용 현황
- ✅ **ServerList** — 레퍼런스로 `createTree` 전환 완료(`ServerList.js fnRenderTree`).
- ⏳ **WS30 USP / WS20 디자인트리** — 다음 단위에서 동일 base 채택 예정(각자 추가기능은 슬롯으로 유지).
- 새 트리 화면은 **처음부터 `createTree` 를 소비**한다.

### 3.6 참고
- 기준 트리(서버 리스트)의 기능 스펙은 [02_로그인_서버리스트.md](02_로그인_서버리스트.md) 참조.

## 4. 스플릿바(Splitter) — 공통 `.u4a-splitter*` + 서버리스트 기준 그립

좌우/3분할 패널을 나누는 **분할 리사이저(스플릿바)** 는 화면마다 새로 만들지 않고, 공통 골격
(`shell.css` 의 `.u4a-splitter*`)과 공통 드래그 동작을 소비하며 **모양(그립)은 서버리스트 기준으로
통일**한다. 트리(§3)와 동일한 단일화 원칙이다.

### 4.1 DOM 계약 (필수)
- 컨테이너 `<div class="u4a-splitter">` 안에 **패널과 바를 번갈아** 배치한다.
  - 2분할: `pane · bar · pane`
  - 3분할(마스터-디테일): `pane · bar · pane · bar · pane`
- 패널 `<div class="u4a-splitter__pane">`, 바 `<div class="u4a-splitter__bar" role="separator">`.
- 컨테이너는 `display:flex; width/height:100%; overflow:hidden`(패널 폭 합이 넘쳐도 페이지로 전파하거나
  세로 공간을 침범하지 않게 클립) — `shell.css` 가 소유한다.
- 패널 폭은 **인라인 `flex`** 로 표현한다: 초기엔 비율(예: 좌측 `0 0 30%`, 가운데 축소형 `0 1 34%`),
  **마지막 패널은 `1 1 auto`**(잔여 채움). 드래그하면 고정 패널이 `0 0 <px>px` 로 바뀐다.
- **초기 고정 px 폭 금지**(반응형, [12번](12_테마_컨버전_전략.md) 7장) — 초기 basis 는 %/유동 단위,
  px 폭은 사용자의 드래그 결과로만 생긴다.

### 4.2 시각 표준 — 서버리스트 기준 그립 (필수)
- 바 폭 11px, 표면색 배경 + 좌우 1px 경계.
- **가운데 그립 핸들**(바 `::before`): 5px×36px 알약(radius 4px), 평소 뮤트색(`--icon-muted` 계열) opacity .5.
- **hover**: 바 배경 → **불투명 `--surface` + `--hover-bg` 틴트 오버레이**(`background-color:--surface` +
  `background-image:linear-gradient(--hover-bg,--hover-bg)`), 그립 → **accent 색 + opacity 1 + 높이 50px** 로 강조.
  ⚠️ 바 hover 에 `background: var(--hover-bg)`(반투명 단색)만 주지 말 것 — 반투명이라 surface 가 사라지고
  **바 뒤 컨테이너 배경이 비치는데, 그 컨테이너가 화면마다 달라(WS20=app-bg, 옵션=투명→body 등) 같은
  토큰인데도 hover 색이 화면마다 달라 보인다.** surface 베이스+틴트로 고정해야 전 화면 동일(2026-06-22).
- 커서 `col-resize`. 색은 전부 의미 토큰만 사용(하드코딩 hex 금지). 치수(11/5/36/50px)는 그립 형태값으로 허용한다.
- ⚠️ **바 배경을 화면별로 override 할 땐 `:hover` 도 같은 특이도로 함께 override 한다(필수).** 공통 hover 는
  `.u4a-splitter__bar:hover { background: var(--hover-bg) }`(클래스 특이도 0,2,0). 화면에서 `#id .u4a-splitter__bar { background: … }`(id 특이도 1,1,0)로 **기본 배경만** 바꾸면 그게 공통 `:hover` 를 이겨 **그 화면만
  hover 색이 사라진다.** → 반드시 `#id .u4a-splitter__bar:hover { background: var(--hover-bg) }` 도 같이 준다.
  (WS20·옵션 팝업에서 이 specificity 함정으로 hover 통일이 깨졌던 사례 — 2026-06-22.)

### 4.3 동작 — 인접 재분배 드래그 + 리사이즈 재클램프 (필수)
- **드래그**: 바를 잡으면(`mousedown`) `body` 커서를 `col-resize` 로, `mousemove` 로 **바 양옆 인접 두 패널
  사이에서만** 폭을 주고받는다(나머지 패널은 불변). 각 패널의 `min-width` 이하로는 줄지 않는다.
- **마지막(유연) 패널 보호**: 오른쪽 이웃이 마지막 `1 1 auto` 패널이면 왼쪽만 px 로 두고 오른쪽은 flex 로
  잔여를 채운다 → 마지막 패널(상세 등)이 짜부되어 사라지지 않는다.
- **창 리사이즈 재클램프**: 창이 줄면 `px` 고정 패널 합 + 바 + 유연패널 최소폭이 컨테이너를 넘지 않도록
  **큰 고정 패널부터 min 까지 자동 축소**한다(`overflow:hidden` 에 잘려 마지막 패널/바가 숨는 것 방지).
  유연/% 패널은 CSS `flex`+`min-width` 로 알아서 줄므로 **px 고정분만** 처리한다. 리스너는 **1회만 바인딩**하고,
  뷰 전환으로 스플릿이 새로 그려져도 `querySelector` 로 그 시점의 스플릿을 처리한다.
- min-width 폴백 120px.
- **더블클릭 = 최초 위치 복귀(공통·전역 자동, 승격 완료)**: 바를 더블클릭하면 인접 패널이 **처음(드래그 전) 폭**으로
  돌아간다. `u4a-ui.js _installGlobalSplitterReset()` 가 모듈 로드 시 1회 자동 설치 — 모든 `.u4a-splitter__bar` 에
  **위임 1개**로 적용(화면별 배선 불필요, 다이얼로그 드래그와 동일 패턴). 최초 폭은 각 패널이 "처음 드래그되기
  직전"(document `mousedown` **capture** → 화면별 드래그보다 먼저 실행)의 인라인 `style.flex`(없으면 `""`=CSS
  기본)를 `data-u4aSplitHome` 에 **1회만** 기록(이미 있으면 안 덮음 → 두 바가 공유하는 패널(3분할 가운데)도 최초값
  유지), 더블클릭 시 바의 prev/next 형제(=인접 패널, 클래스 무관)를 그 값으로 복귀. 드래그/재클램프(아직 화면별
  복제)와 달리 **이 동작은 단일 전역 헬퍼로 이미 승격됨** → 새 분할 화면은 바에 `.u4a-splitter__bar` 만 붙이면 자동.

> **필수 엣지 케이스 — 최대화 후 restore 시 바/패널이 숨는 문제**
> 브라우저(창) **최대화 상태에서 스플릿바를 오른쪽 끝까지 드래그**하면 좌측 패널이 큰 `px` 폭으로 고정된다.
> 이 상태로 창을 **restore(축소)** 하면, 고정 px 폭 합 + 바 폭이 줄어든 컨테이너를 초과해 `overflow:hidden` 에
> 뒤쪽 패널·바가 잘려 **사라진 것처럼 보인다.** → 위 "창 리사이즈 재클램프"가 이를 막는 **필수** 처리다.
> 새 분할 화면은 반드시 이 재클램프를 바인딩한다.
> 레퍼런스(동일 패턴): ServerList `_bindSplitterResizeClamp`, WS20 `_bindWs20SplitResizeClamp`.

  `--sl-*` 그립 override 제거 — 그게 화면 간 바 색 불일치 원인이었음. 의미 토큰으로 통일.) 드래그/클램프는

  (surface-raised+하단 보더)** 으로 본문과 분리(`_ensureStyle .u4aOptSecHdr`/`.u4aOptSecBody`). (옛날엔 좌우·바를

- ⚠️ **남은 이원화**: 그립 **스킨·더블클릭 리셋은 단일화 완료**됐으나, **드래그/재클램프 JS 로직은 여전히 화면별

- ⏳ **통일 방향(승격)**: 4.3 드래그/재클램프를 **공통 JS 헬퍼 1벌**로 올려 전 화면이 옵션 없이 동일 동작을 소비.
- 새 분할 화면은 **처음부터 `.u4a-splitter*` + 공통 그립 소비**(자체 색/그립 스킨 금지) + 공통 드래그를 소비한다.

## 5. 다국어(i18n) / 텍스트

화면에 보이는 **모든 문구는 SQLite 메시지 클래스의 키**에서 가져온다. 문자열을 소스에 직접 적지 않는다
(하드코딩 금지). 컨버전·신규 화면 공통 규칙이며, 메시지 키 사용 원칙은 [02번](02_로그인_서버리스트.md)·
[15번](15_공통_입력UX_가이드.md)과 동일하다.

### 5.1 공통 추출 함수 (필수)
- **저수준**: `WSUTIL.getWsMsgClsTxt(LANGU, MSGCLS, MSGNR, p1, p2, p3, p4)` — SQLite 에서 언어/클래스/번호로
  텍스트를 조회하고 `&1`~`&4`(및 `&`) 플레이스홀더를 `p1`~`p4` 로 치환한다.
- **래퍼**: `fnGetMsgClsText(MSGCLS, MSGNR, p1..p4)` — `LANGU` 를 워크스페이스 언어로 해석한 뒤 위 저수준 함수를
  호출한다. 래퍼 진입점은 화면마다 다를 수 있으나(예: ServerList `MessageClassText.fnGetMsgClsText`,
  WS `oAPP.common.fnGetMsgClsText`) **모두 `WSUTIL.getWsMsgClsTxt` 로 위임**된다 — 새로 만들지 말고 그 화면 래퍼를 쓴다.
- 화면 코드는 위(저수준/래퍼) 중 하나만 쓴다. **리터럴 문자열을 DOM 에 직접 넣지 않는다.**

### 5.2 언어 기준
- 라벨 언어 = **워크스페이스(접속 사용자) 언어** — `parent.process.USERINFO.LANGU`(있으면 최우선),
  WS UI 언어는 `WSUTIL.getWsLanguAsync()`. **백엔드 `serverInfo.LANGU` 가 아니다.**
- 메시지 클래스 예: `ZMSG_WS_COMMON_001`(번호 M0271 등), `/U4A/CL_WS_COMMON`(번호 A68/A69 등).

### 5.3 금지
- 하드코딩 문자열, **영문(또는 임의) 폴백 기본값**, 완료 토스트 등 **임의 문구 생성**, **가짜/추측 키** 사용 금지.
- 기존 키가 있으면 **그대로 재사용**한다(같은 의미의 키 중복 생성 금지).

### 5.4 신규 텍스트가 필요할 때 (작업 흐름)
화면에 **기존 키가 없는 새 문구**가 필요하면:
1. 소스에는 **공통 추출 함수 호출만** 넣는다(하드코딩·자리 임시문구 금지). 그 자리에서 임의 키를 확정하지 않는다.
2. 작업 중 발생한 **신규 텍스트를 모았다가, 작업 끝에 "신규 항목만" 목록으로 제출**한다(아래 양식).
3. 사용자가 그 목록을 **번역하여 SQLite DB 에 추가/업데이트 → 앱 재실행**한다.

**신규 텍스트 제출 양식**(예시):

| # | 메시지클래스(제안) | 번호(제안) | 한국어 원문 | 플레이스홀더 | 사용 위치(파일 · UI) |
|---|---|---|---|---|---|
| 1 | /U4A/CL_WS_COMMON | (신규) | 서버를 선택하세요 | – | ServerList.js · 빈 상태 안내 |
| 2 | /U4A/CL_WS_COMMON | (신규) | &1 개 항목이 선택됨 | `&1`=개수 | (예) 다중선택 푸터 |

- 클래스/번호는 **제안**이며, 최종 확정은 사용자(번역·DB 입력) 몫이다.
- 플레이스홀더가 있으면 `&1`~ 표기와 의미를 함께 적는다.

## 6. 테이블(Table) — 공통 `.u4a-table` + 컬럼 정렬/필터 메뉴 (서버리스트 기준)

목록/그리드는 **공통 테이블 컴포넌트**를 소비한다. 화면마다 표 스타일·정렬·필터를 새로 만들지 않는다.

### 6.1 단일 출처 (필수)
- 마크업: `.u4a-table-wrap > table.u4a-table`(`shell.css`). 밀도 축소는 `.u4a-table--compact`(행 빽빽하면 빼고 기본 `.u4a-table`).
- 헤더 sticky, zebra=행 `data-odd="true"`, hover, **선택=행 `aria-selected="true"`(좌측 accent 바)**, 빈 결과=`tr.u4a-table__nodata`, 포커스 링.
- 좁아지면 카드(타일) 뷰: `.u4a-table-wrap[data-view="card"]`(컨테이너쿼리 미지원 → 화면 JS 가 ResizeObserver 로 `data-view` 토글).
- `.u4a-table*` 는 전부 `shell.css` 공통. 화면별 확장(컬럼 폭·정렬메뉴 헤더 등)만 `.u4a-c-*`/스코프로 얹는다.

### 6.2 컬럼 헤더 정렬/필터 메뉴 — 서버리스트 기준 (필수)
원본 UI5 `sap.ui.table` 의 `sortProperty`/`filterProperty`(헤더 클릭 시 정렬·필터 메뉴)는 **서버리스트 `fnOpenColumnMenu` 패턴으로 통일**한다.
- 정렬·필터 가능한 `th` 에 `.u4a-th--menu`(클릭) + 내부 `.u4a-th__inner`(라벨 `.u4a-th__label` + 표시자 `.u4a-th__ind`).
- 클릭 → 공통 `.u4a-menu.u4a-colmenu`. **구성·순서(서버리스트 동일)**: ① 필터 input(`.u4a-colmenu__filter > .u4a-input`) ② 구분선 ③ 오름차순 정렬 ④ 내림차순 정렬 ⑤ 구분선 ⑥ 필터 초기화(이 컬럼).
- **필터**: 표시 텍스트 기준 contains(대소문자 무시), 여러 컬럼 **AND**, Enter/blur 시 적용. **정렬**: 단일 컬럼 `localeCompare(…,{numeric:true})`, 같은 방향 재클릭 시 해제. 활성 정렬 항목=`.u4a-menu__item[data-active="true"]`.
- **표시자**(`.u4a-th__ind`): 정렬 걸린 컬럼=화살표(arrow-up/down), 필터 걸린 컬럼=깔때기(funnel). 걸린 컬럼만 노출.
- **원본 배열(검색결과 등)은 보존**하고 화면에 그릴 **뷰만 필터→정렬로 파생**(비파괴). 헤더는 매 렌더 재구성(표시자 갱신).
- 메시지 키: 오름/내림 정렬=`ZMSG_WS_COMMON_001` 810/811, 필터 값(placeholder)=`/U4A/CL_WS_COMMON` A68, 필터 초기화=A69.
- ⚠️ `.u4a-th__*`·`.u4a-colmenu*` CSS 는 현재 **서버리스트 전용(`serverlist.css`)** — 다른 창은 스코프로 복제(공통 `.u4a-menu`/`.u4a-menu__item` 은 `shell.css`). **향후 `shell.css` 승격 권장.** showModal 다이얼로그 안 테이블이면 메뉴를 **다이얼로그(top-layer) 안에 append**(밖이면 모달에 가림) + `getBoundingClientRect`+`position:fixed` 로 헤더 아래 배치.

### 6.3 적용 현황
- 레퍼런스: 서버리스트(`ServerList.js` `fnOpenColumnMenu`/`_buildServerView`).
- 소비: errMsgPopup, App Search Help F4 팝업(탭1 결과 테이블).

## 7. 원칙 요약

- **모든 화면 = Bootstrap 베이스 + tokens.css 테마.**
- **모달/메시지 박스 = 헤더 48px · 푸터 48px · 드래그 · 리사이즈.**
- **트리 = 공통 베이스 `U4AUI.createTree` 소비** (role 미부착 · 토글 WS20식 · 들여쓰기 `--u4a-tree-depth` 표준 · 추가기능은 slot/hook 확장).
- **스플릿바 = 공통 `.u4a-splitter*` + 인접 재분배 드래그 · 리사이즈 재클램프 소비** (모양은 서버리스트 기준 그립으로 통일, 초기 고정 px 폭 금지).
- **테이블 = 공통 `.u4a-table` 소비 + 컬럼 정렬/필터 메뉴는 서버리스트 기준** (`.u4a-th--menu`→`.u4a-colmenu`: 필터 input · 오름/내림 정렬 · 초기화 / 필터 contains·AND, 정렬 단일·numeric, 원본 보존+뷰 파생).
- **다국어 = 모든 문구를 SQLite 메시지 키 + 공통 추출 함수(`WSUTIL.getWsMsgClsTxt`/`fnGetMsgClsText`)로** (하드코딩·임의키 금지, 신규 텍스트는 목록 제출 → 사용자 번역·DB 입력).
- 공통 자산(Bootstrap, tokens.css, shell.css, u4a-ui.js)을 **소비**하고 화면마다 새로 만들지 않는다.
