# 15. 공통 입력(Input) UX 가이드

> 모든 화면의 `<input>` 은 **동일한 UX** 를 따른다. 새 입력칸을 만들 때(또는
> 다른 PC/세션에서 AI 가 작업할 때) 이 문서의 규칙을 그대로 적용한다.
> 단일 출처(SSOT)는 테마의 `shell.css` + `u4a-ui.js` 이며, 화면별로 마크업/스타일을
> 새로 만들지 말고 **공통 컴포넌트를 소비**한다. (UI5 `sap.m.Input` 의
> `showClearIcon` / number 스피너 등을 HTML5 로 대체한 결과물)

---

## 0. TL;DR — 새 입력칸 만들 때 체크리스트

1. 입력은 `.u4a-field` 래퍼로 감싸고, 값이 있을 때만 보이는 **X(clear) 버튼**을 단다.
2. `type="number"` 입력의 **위/아래 스피너는 항상 제거**한다 (CSS 전역 처리됨 — 추가 작업 불필요).
3. 마크업/동작은 직접 구현하지 말고 **공통 헬퍼**(`U4AUI.attachClear`)와
   **공통 클래스**(`.u4a-field`, `.u4a-field__input`, `.u4a-field__clear`)를 사용한다.
4. 트레일링 아이콘 개수에 맞춰 래퍼에 `data-trail="1"`(X만) / `"2"`(X + value-help·비번토글 등)를 준다.

## 1. 단일 출처(SSOT) 파일

| 파일 | 역할 |
|------|------|
| `www/ws30/ws10_20/theme/u4a-ui.js` | **`U4AUI.createField(opts)` 입력 팩토리(권장 — sap.m.Input 대응)** + `attachClear/attachSuggest/createSelect` 동작 헬퍼. `window.U4AUI` 전역 |
| `www/ws30/ws10_20/theme/shell.css` | `.u4a-input`, `.u4a-field*` **구조/노출 규칙**(크기·테두리폭·radius·포커스·readonly hover 무력화) |
| `www/ws30/ws10_20/theme/bootstrap-skin.css` | 입력 **색의 단일 출처**(shell 뒤 로드해 이김): 편집=`--surface-raised`+`--divider`, readonly/disabled=`--app-bg`. ⚠️색 바꾸려면 여기서(shell.css 만 고치면 안 먹힘) |

### 1.1 입력 생성은 `U4AUI.createField` 로 (필수 권장)
화면마다 `createElement("input")`+클래스+클리어/서제스트를 직접 짜지 말고 **`U4AUI.createField(opts)` 한 번**으로 생성한다(생성+동작 단일화 → 부분수정 두더지잡기 방지).
- `opts`: `type`(text/password/textarea/combo/select)·value·placeholder·id·readOnly·disabled·rows·width·`clear`·`suggest`(+onPick)·`f4`(+f4Icon)·`upper`·onChange/onEnter/onInput·items/onOpen(combo)·className/inputClassName.
- 반환: `{ el, input, getValue, setValue, setReadOnly, setValueState, setItems, focus }`. `el` 을 폼에 붙이고 id/`data-*` 는 `input` 에 직접 부여(렌더 로직이 getElementById/`[data-*]` 로 접근).
- 인라인 행(입력+버튼)에서는 `.u4a-field` 가 flex 자식이므로 그 행 CSS 에 `.u4a-field{ flex:1 1 auto }` 만 더한다(스택형은 불필요).
- 이관 현황: WS30 USP 완료, 나머지 화면 순차 이관(WS20 속성/Find·WS10·Login·ServerList·F4팝업).

화면 스크립트(Login.js, ServerList.js, WS20/WS30 등)는 위 파일을 로드한 뒤 **createField 로 소비만** 한다.

## 2. 클리어(X) 버튼 — "값 있을 때만"

### 2.1 마크업
```html
<div class="u4a-field" data-trail="1">                      <!-- 래퍼(상대좌표) -->
  <input class="u4a-input u4a-field__input" type="text">    <!-- 우측 패딩 자동 -->
  <button type="button" class="u4a-field__clear"
          title="Clear" aria-label="Clear" tabindex="-1">×</button>
</div>
```
- 아이콘은 input "안쪽" 우측에 `absolute` 로 겹친다 (밖에 따로 두지 않는다).
- 버튼 내용은 Font Awesome `xmark` 아이콘 사용 (`ICON.clear = _fa("xmark")`).
- `tabindex="-1"` — Tab 이동 시 건너뛴다.

### 2.2 노출 규칙 (CSS가 처리)
- `.u4a-field__clear` 는 기본 `display:none`.
- 래퍼에 `data-filled="true"` 가 붙은 동안에만 노출 → **값이 있을 때만 X 표시**.
- `data-filled` 토글은 `attachClear` 가 자동으로 해 준다(직접 만지지 않는다).
- **읽기전용/비활성 입력엔 X 를 띄우지 않는다**(공통 CSS `shell.css .u4a-field__input:read-only ~ .u4a-field__clear{ display:none }`). 모드(Display↔Change)로 readonly 가 토글되는 필드는 `createField({ clear:true })` 로 X 를 정적으로 달아두면 readonly 일 때 자동으로 숨겨진다(화면이 모드별로 붙였다 떼지 않는다).

### 2.3 동작 연결 (JS)
```js
window.U4AUI.attachClear(oInput, oClearBtn /*, fnAfterClear */);
```
- 타이핑 등 값 변화(`input` 이벤트)마다 노출 상태를 동기화.
- X 클릭 시: 값 비우기 → `input` 이벤트 재발화(모델/검증/자동완성 동기화) → 입력에 포커스 유지.
- `mousedown` 에서 `preventDefault` 라 클릭해도 포커스가 빠지지 않는다.
- 프로그램으로 값을 set 한 뒤에는 반환된 `_sync()` 를 호출해 노출 상태를 재계산.

### 2.4 트레일링 슬롯이 둘 이상일 때
오른쪽에 value-help(▾)·비밀번호 토글(👁) 같은 아이콘이 더 있으면:
- 래퍼에 `data-trail="2"`.
- value-help/비번토글은 `.u4a-field__vh` / `.u4a-field__pw` (맨 우측).
- X(clear)는 자동으로 그 **왼쪽**에 배치된다(CSS가 위치 계산).

## 3. number 입력 스피너 제거

`type="number"` 의 네이티브 위/아래 스피너는 clear(X)와 겹치고 UX 통일성을 해쳐
**모든 화면에서 제거**한다. `shell.css` 에 전역 규칙이 있으므로 별도 작업이 필요 없다.

```css
.u4a-input[type="number"] {
    appearance: textfield;
    -moz-appearance: textfield;
}
.u4a-input[type="number"]::-webkit-outer-spin-button,
.u4a-input[type="number"]::-webkit-inner-spin-button {
    -webkit-appearance: none;
    margin: 0;
}
```
- 값 증감은 직접 타이핑으로 한다(스피너 미제공).
- 숫자 입력 제한이 필요하면 `maxLength`/`inputmode="numeric"` 등을 병행한다.

## 3.5 ValueState(검증) 메시지 — "포커스일 때만" (UI5 동일)

UI5 `sap.m.Input` 의 `valueState`/`valueStateText` 를 HTML5 로 대체한 규칙.
**오류 테두리(빨강)는 항상 유지**하고, **메시지 팝업은 입력칸에 포커스가 있을 때만**
뜬다(마우스/키보드로 포커스 → 표시, 커서가 빠지면(blur) 즉시 숨김). SAPUI5 와 동일하며
`.u4a-field__msg` 를 쓰는 **모든 팝업에 자동 적용**(별도 배선 불필요).

### 3.5.1 마크업 (행 = `.u4a-form__row`, 메시지는 그 직계 자식)
```html
<div class="u4a-form__row">
  <label class="u4a-label u4a-label--required">메서드 이름:</label>
  <div class="u4a-field" data-trail="1">
    <input class="u4a-input u4a-field__input">
    <button class="u4a-field__clear" tabindex="-1">…</button>
  </div>
  <div class="u4a-field__msg"></div>            <!-- 비어있으면 안 보임 -->
</div>
```

### 3.5.2 노출 규칙 (CSS — `shell.css`, 전역)
- `.u4a-field__msg` 기본 `display:none`.
- `.u4a-form__row:focus-within .u4a-field__msg:not(:empty)` 일 때만 `display:inline-flex`.
  → 행 안의 입력칸/트레일링 버튼에 포커스가 있고, 메시지 텍스트가 있을 때만 노출.
- 빨간 테두리는 `.u4a-input[data-vs="error"]` 가 별개로 항상 유지(포커스 무관).
- 메시지는 `position:absolute; top:100%` 로 입력칸 아래에 **떠서** 표시(행 간격 영향 없음).

### 3.5.3 동작 연결 (JS — 화면 코드)
```js
// 오류일 때: 입력에 빨간 테두리 + 메시지 텍스트 채우기. (포커스면 자동으로 메시지가 뜬다)
oInput.setAttribute("data-vs", "error");
oMsg.textContent = "…메시지…";
// 정상으로 되돌릴 때:
oInput.removeAttribute("data-vs");
oMsg.textContent = "";
```
- 검증 실패 직후 메시지를 **바로** 보이게 하려면 그 입력칸에 `focus()` 를 준다
  (포커스가 없으면 빨간 테두리만 보이고 메시지는 다음 포커스 때 뜬다 — UI5 동일).
- value-state 메시지와 자동완성(`attachSuggest`)이 동시에 뜨면 **메시지 위 / 제안 목록 아래**로
  자동 스택된다(`attachSuggest` 가 같은 행의 메시지 높이만큼 목록을 내려 겹침 방지).
- `information` 변형(예: 로그인 CapsLock)은 `.u4a-field__msg[data-vs="information"]` 으로
  빨간 X 대신 파란 info 아이콘.

## 3.6 읽기전용/표시전용 필드 — 활성/비활성 시각 구분 (공통)

편집 가능 필드와 비활성 필드는 **한눈에 구분**되어야 한다(전 화면 공통). 화면별로 따로 칠하지 않는다.
**핵심 컨셉**: 둘 다 테두리(필드처럼)는 있되, **활성=올라온 배경+또렷한 테두리 / 비활성=평평 배경+옅은 테두리+muted** 로 강약 구분.

- **편집 가능(활성)**: `.u4a-input`/`.u4a-combo` = `--surface-raised` 배경(패널보다 올라옴) + **또렷한 `--divider` 테두리** + 일반 글자 → "여기 입력".
- **비활성(읽기전용/disabled/--display)**: **평평한 `transparent` 배경** + **옅은 `--line` 테두리** + `color` muted(`--sl-muted`/`--text-muted`) + opacity:1 + hover/포커스 강조 없음 + 기본 커서. 필드로는 보이되 뒤로 물러남 → "편집 불가". (입력은 `disabled`(opacity 흐림) 대신 **`readOnly`** 권장 — 값 선택/복사 가능. 콤보 비활성 화살표는 muted 로 표시.)
- ⚠️ **색·보더 단일 출처 = `bootstrap-skin.css`** (`.u4a-input:disabled,[readonly],.u4a-input--display` / `.u4a-combo[aria-disabled],.is-disabled`) — `shell.css` 보다 **늦게 로드 + 같은 명시도라 이긴다**. 바꾸려면 bootstrap-skin 을 고친다(shell.css 만 고치면 안 먹힘). shell.css 대응 규칙도 같게 맞춰 둠.
- ⚠️ **시행착오(2026-06-23)**: ① 구 `--app-bg` 어두운 박스 readonly → 다크에서 "어두운 박스=입력칸" 직관과 충돌해 readonly 가 활성처럼 보임. ② 그래서 '테두리 0 평문'으로 갔더니 너무 휑하고 활성 테두리(흐린 `--line`)도 약해 활성도 테두리 없어 보임. ③ **정답 = 활성 또렷한 `--divider` + 비활성 옅은 `--line` 유지**(둘 다 테두리, 강약 구분). 비활성 배경은 `--disabled-bg`(다크=`--surface`)·`--sl-surface-2`(hover톤)·`--app-bg`(어두운박스) 셋 다 금지, `transparent`.

## 3.7 hover / focus 강조 — 편집 가능 input·combo 공통 (전 화면 글로벌)

편집 가능한 `.u4a-input` / `.u4a-select` / `.u4a-combo` 는 **마우스 오버 시 테두리가
`--text-muted` 로 강조**되고, **포커스 시 `--accent` 테두리 + 옅은 halo** 가 뜬다
(UI5 `sap.m.Input` 의 hover/focus 상태 대체). 이 규칙은 `shell.css` **한 곳의 글로벌
셀렉터**(`.u4a-input:hover` / `.u4a-combo:hover` / `:focus-visible`)로만 정의되어,
화면별 분기 없이 전 화면(로그인·팝업·WS10/20/30·ServerList)에 동일 적용된다.

- 화면별 CSS 에서 input/combo 의 `:hover` 테두리를 **따로 정의하거나 무력화하지 않는다**
  (그 순간 화면마다 강조가 달라져 통일이 깨진다). 강조 ON/OFF 는 이 글로벌 규칙에서만 바꾼다.
- 읽기전용/비활성(`:read-only` / `--display` / `[aria-disabled]`)은 hover 강조를 주지 않는다
  (편집 불가 신호 유지 — §3.6).
- 콤보 휴지 배경/테두리도 입력칸과 동일(`--surface-raised` + `--divider`)이라 hover 변화 폭까지
  input 과 똑같이 보인다(콤보만 어둡던 옛 `--sl-bg`/`--surface` 는 폐기).
- 별도 BrowserWindow 팝업(F4 등)에서 강조가 안 보이면 코드 차이가 아니라 그 창의 셸 CSS
  캐시다 — 리로드/재빌드하면 동일하게 적용된다.

## 4. 적용 예시 (ServerList 편집 팝업)

`www/ServerList_v2/ServerList.js` 의 `_wrapClear(oInput)` 헬퍼가 이 가이드를
캡슐화한 예시다. 동적으로 input 을 만드는 화면은 이런 헬퍼를 두고 재사용한다.

```js
// 입력을 .u4a-field 래퍼로 감싸고 "값 있을 때만" X(clear) 버튼을 단다.
function _wrapClear(oInput) {
    oInput.classList.add("u4a-field__input");
    const oWrap = _el("div", "u4a-field");
    oWrap.dataset.trail = "1";
    const oClear = _el("button", "u4a-field__clear");
    oClear.type = "button";
    oClear.title = "Clear";
    oClear.setAttribute("aria-label", "Clear");
    oClear.tabIndex = -1;
    oClear.innerHTML = ICON.clear;                 // _fa("xmark")
    oWrap.append(oInput, oClear);
    window.U4AUI.attachClear(oInput, oClear);
    return oWrap;
}
```
정적 HTML 마크업으로 만드는 화면(Login.js)은 위 §2.1 의 마크업을 직접 쓰고
초기화 시 `attachClear` 를 호출한다.

## 5. 원칙 요약

- **모든 입력칸은 동일한 clear(X) UX 를 가진다 — 값이 있을 때만 노출.**
- **검증 메시지(`.u4a-field__msg`)는 포커스일 때만 뜬다(UI5 valueState 동일) — 빨간 테두리는 항상 유지.**
- **number 스피너는 어디서도 보이지 않는다.**
- 모양/동작은 화면마다 새로 만들지 말고 `shell.css` + `u4a-ui.js` 공통 자산을 소비한다.
- 새 트레일링 아이콘(검색/달력 등)이 필요하면 `.u4a-field__*` 패턴을 확장해
  공통 파일에 추가하고, 화면은 그것을 소비한다.
