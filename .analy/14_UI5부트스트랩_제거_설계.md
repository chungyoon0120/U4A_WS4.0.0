## 1. 핵심 전략 — "오버라이드 스크립트" (WS3.0 검증)

원본 UI5 파일(`ws_fn_01.js`, `ws_common.js`, `ws_main.js`, `ws_fn_02.js`…)을 **갈아엎지 않는다.** 대신 `library-preload.js` 의 로드 목록 **맨 끝**에 `ws_html5_*` 스크립트를 추가해, UI5를 쓰는 렌더/모델 함수를 **HTML5 구현으로 덮어쓴다(override)**. eval 로드 순서상 나중 정의가 이긴다.

```
... ws_fn_01~05, ws_main.js (원본, UI5 함수 포함) ...
└─ ws_html5_shell.js      ← 모델 shim + 부팅 + WS10 셸/렌더/네비/푸터 override
└─ ws_html5_ws20.js       ← WS20 셸(트랜잭션 툴바 + 3분할)
└─ ws_html5_ws20_tree.js  ← WS20 UI 트리 (TreeTable → HTML5 재귀 트리)
└─ ws_html5_ws20_data.js  ← WS20 트리 데이터 파이프라인(/getAppData → /zTREE)
└─ ws_html5_ws20_attr.js  ← WS20 속성 패널
└─ ws_html5_ws20_prev.js  ← WS20 Preview 호스트(iframe 삽입; 내부 UI5 KEEP)
```

> 장점: 원본 로직(검증·이벤트·서버호출) 보존, diff 최소, 롤백은 override 스크립트 제거로 즉시. 현재 WS4.0 의 `js/ws10_html.js`(1단계 산출물)가 곧 `ws_html5_shell.js` 의 WS10 렌더 부분이 된다 → **수렴/확장**한다.

## 3. 키스톤 — 모델 shim & 부팅 (ws_html5_shell.js, WS3.0 검증 코드)

UI5 제거의 척추는 **JSONModel → 경로기반 상태객체**. 앱 전역이 `APPCOMMON.fnSetModelProperty("/A/B", v)` / `fnGetModelProperty("/A/B")` 만 쓰므로 그 두 개만 충실히 구현하면 13개 파일 호출부가 **그대로** 동작한다(§stage1 에서 우려한 `AppNmInput`/모델 결합이 여기서 풀린다).

```js
// 1) 경로기반 상태 모델 (oAPP.attr.oModel) — getProperty/setProperty/oData/setData/refresh
function _createModel() { var _data={};
  return {
    get oData(){return _data;}, set oData(v){_data=v||{};},
    setData(o){_data=o||{};}, getData(){return _data;},
    getProperty(p){ /* "/A/B" 경로 탐색 */ },
    setProperty(p,v){ /* 경로 set 후 fnRenderFooterMsg() 등 부분 렌더 */ },
    refresh(){}              // 디자인 코드가 호출 → no-op(크래시 방지). WS20 트리가 래핑.
  };
}
// 2) BusyDialog 호환 더미 (setBusy 가 open/close 호출) → parent.setDomBusy 위임
oAPP.fn.fnCreateDummyBusy = function(){ return { open(){parent.setDomBusy("X");}, close(){parent.setDomBusy("");}, isOpen(){...}, setText(){}, setTitle(){} }; };
// 3) 모델 get/set override (구 sap.ui.getCore().getModel().get/setProperty)
oAPP.common.fnGetModelProperty = (p)=> oAPP.attr.oModel ? oAPP.attr.oModel.getProperty(p) : undefined;
oAPP.common.fnSetModelProperty = (p,v)=> { if(oAPP.attr.oModel) oAPP.attr.oModel.setProperty(p,v); };
// 4) fnOnInitModelBinding override → oAPP.attr.oModel = _createModel(); setData(메타)
// 5) fnGetWsMsgModelData override → getModel() 대신 fnSetModelProperty("/WSLANGU",...)
```

추가 override(메시지/렌더 계열): `checkWLOList`, `fnIsStaff`, `setWSHeadText`, `fnSetBusyLock`, `fnShowFloatingFooterMsg`/`fnHideFloatingFooterMsg`/`fnRenderFooterMsg`, `fnOnInitP13nSettings`(경량/no-op), `fnOnInitRendering`/`fnOnInitRenderingWS10_HTML5`, `fnOnEnterDispChangeMode`(서버 흐름 보존), `fnOnMoveToPage`(NavContainer.to → showPage), `fnGetWs10AppInputDom`(HTML input 핸들) 등 — 전부 WS3.0 `ws_html5_shell.js` 에 구현돼 있어 **이식**하면 된다.

## 4. 진행 단계 (각 단계 = 사용자 Electron 검증 게이트)

> 환경상 나(에이전트)는 Electron 실행 불가 → **단계마다 사용자가 로그인/조회로 검증**. 단계는 작게.

- **S0 (완료, 1단계)**: `ws10_html.js` 로 WS10 화면만 HTML 렌더. UI5 부팅은 유지. → *로그인 시 HTML WS10 표시 확인됨.*
- **S1 — 모델 shim + WS10 로직 연결**: `ws10_html.js` → `ws_html5_shell.js` 로 승격. §3 모델 shim 이식 + WS10 의 메뉴/트랜잭션/검색을 **원본 핸들러(ev_App*, fnHmws, F4, suggestion)** 에 연결(이제 `fnGetModelProperty`/`AppNmInput` DOM 핸들로 동작). **이 단계까지는 UI5 부트스트랩 유지**(WS20 가 아직 UI5라 안전). → 검증: WS10 버튼/검색/메뉴 실제 동작.
- **S2 — 부팅 디커플(부트스트랩 제거의 핵심)**: 매핑 A·C·D·B 적용 — `index.js` 부트스트랩 주입 제거, `library-preload` 즉시 실행 + override 등록, `ws_main.fnWsStart` attachInit/BusyDialog/UIUpdated 정리, HTML5 `vw_main`. → 검증: **UI5 CDN 로드 없이** 로그인→WS10 정상. (이 시점에 WS20 진입은 아직 미구현이면 막아두거나 placeholder)
- **S3 — WS20 셸/트리/데이터/속성**: `ws_html5_ws20*.js` 4종 이식. TreeTable→HTML5 트리, 속성 패널, /getAppData 파이프라인. → 검증: 앱 조회→WS20 진입, 트리/속성 표시.
- **S4 — WS20 Preview 호스트**: `ws_html5_ws20_prev.js`. iframe 호스트만 HTML5, **내부 UI5 KEEP**. Preview 는 부팅 preload 금지(크래시 전례) → WS20 진입 후 지연 로드. → 검증: 미리보기 렌더/줌.
- **S5 — 정리**: 잔여 `sap.*`(BusyDialog 클래스/아이콘/MessageBox) 제거 가드, 팝업(별도 창) 단위는 후속.

부트스트랩이 **실제로 빠지는 시점은 S2**. S1 까지는 UI5와 공존(안전).

## 5. 리스크 & 가드

## 6. 한 줄 요약

> **원본을 두고 `ws_html5_*` override 로 UI5 렌더·모델을 HTML5로 덮어쓴 뒤, `index.js` 의 부트스트랩 주입과 `attachInit` 진입을 걷어낸다. 모델은 경로기반 상태객체 shim 으로 대체해 13개 호출부를 그대로 살린다. WS20 Preview iframe 의 UI5 만 KEEP. 부트스트랩은 S2 에서 빠지고, 모든 단계는 사용자 Electron 검증으로 넘어간다.**  (WS3.0 에 완성형 코드 존재 — 이식 중심)
