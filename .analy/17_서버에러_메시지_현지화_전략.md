# 17. 서버 예외 메시지 — 클라이언트 언어 현지화 전략

> 대상: HTML5 클라이언트가 서버(ABAP `/U4A/WS_ROOT` → 각 FORM) 호출 후 받는 **예외/오류 응답**을
> **사용자(클라이언트) 언어**로 출력하기 위한 표준. 분석 사례 = `SET_MIME_CRUD`(MIME 폴더/파일 CRUD).

---

## 0. 한 줄 요약 (★전제: 서버 ABAP 수정 불가)
서버는 **메시지를 서버 로그인 언어로 렌더한 텍스트**(RETMSG, 또는 SCRIPT 안 `parent.showMessage(sap,…,'텍스트')`)로만 내려준다. **서버를 못 고치므로**, 클라이언트가 **그 텍스트를 로컬 메시지 DB로 역매핑(text→키)해서 자기 언어로 다시 렌더**한다.
근거: 클라는 `www/MSG/.../MESSAGE_CLASS.db` 에 `/U4A/MSG_WS`·`ZMSG_WS_COMMON_001` 을 **여러 언어로 보유** → 서버가 어떤 언어로 렌더했든 그 언어 텍스트로 **키를 역추적**한 뒤, **클라 언어로 `fnGetMsgClsText` 재렌더**가 가능하다.

---

## 1. 백엔드 소스 확인 방법 (재현 절차)
서버 호출 마지막 패스가 예: `/SET_MIME_CRUD` 이면
1. 대표 서브루틴 풀 **프로그램 `/U4A/WS_ROOT`** 에서 동명 `FORM SET_MIME_CRUD` 를 찾는다.
2. 그 FORM 이 호출하는 `PERFORM ... IN PROGRAM /U4A/WSxxxxxx` 의 대상(여기선 `/U4A/WS000002`)으로 들어간다.
3. 그 안의 `FORM SET_MIME_CRUD` → 다시 `_SET_MIME_CREATE` → `_SET_MIME_CREATE_FOLDER`/`_FILES`, `_SET_MIME_DELETE` 로 따라간다.
(SAP ADT MCP `sap_adt_get_program_source` / `sap_adt_get_source` 로 소스 열람.)

---

## 2. 현재(잘못된) 응답 패턴 — `SET_MIME_CRUD` 실측
서버는 **두 가지 형태**로 오류를 내려주며, **둘 다 메시지를 서버 언어로 이미 렌더**해 보낸다.

**(A) SCRIPT-임베드형** — `RTMSG=''` + `SCRIPT` 에 JS 문자열을 박아 클라가 `eval` 하게 함.
```abap
GS_RETURN_JS-RETCD = 'E'.
GS_RETURN_JS-RTMSG = ''.
CONCATENATE `lf_createMimeCts();parent.showMessage(sap, 20, 'E', '` LV_MSG `');`
  INTO GS_RETURN_JS-SCRIPT.        " LV_MSG = MESSAGE Exxx(/U4A/MSG_WS) INTO LV_MSG (서버언어 렌더)
```
**(B) 텍스트형** — `{ "RETCD":"E", "RETMSG":"<서버언어 텍스트>" }`.
```abap
CONCATENATE `{ "RETCD" : "E", "RETMSG" : "` LV_MSG `" }` INTO LV_JSON.
```

### 2.2 선행검사 서브폼 케이스(SET_MIME_CRUD 진입 직후, **C/D 공통**)
`SET_MIME_CRUD` 는 `CASE TRCOD`(생성/삭제) **이전에** 패키지·CTS 검사를 무조건 수행한다 → 아래 오류는 **생성과 삭제 모두**에서 발생.

| # | 케이스 | FORM(프로그램) | 반환 | 메시지 | 타입 | CTS |
|---|--------|----------------|------|--------|------|-----|
| 9 | 패키지 미입력 | `_CHK_PACK_EXISTS`(WS000001) | RTMSG+SCRIPT | **050** "Package & is required." | E | — |
| 10 | 패키지 미존재 | `_CHK_PACK_EXISTS` | RTMSG+SCRIPT | **073** "Package &1 does not exist." | E | — |
| 11 | Transport Layer 없음 | `_CHK_PACK_TRANSLAYER`(WS000004) | RTMSG+SCRIPT | **231** | E | — |
| 12 | Transport Layer 미존재 | `_CHK_PACK_TRANSLAYER` | RTMSG+SCRIPT | **232** | E | — |
| 13 | CTS 미존재/처리불가 | `_CHK_CTSNO`(WS000004) | RTMSG(**193**)+SCRIPT(**213** W, `lf_createMimeCts`) | W | 필요 |
| 14 | CTS Target System 불일치 | `_CHK_CTSNO` | RTMSG(193)+SCRIPT(**395** W) | W | 필요 |
| 15 | 요청 Task Lock | `_CHK_CTSNO` | RTMSG+SCRIPT | (LV_MSG, 가변) W | — |

> ★두 가지 핵심: ①**표시 권위 = SCRIPT**. 원본은 `eval(SCRIPT)` 로 그 안의 `showMessage(타입,텍스트)` 를 실행한다(타입 **W**도 있음). 13·14 처럼 RTMSG(193)와 SCRIPT(213/395)가 **다를 때** 사용자가 보는 건 SCRIPT 쪽. 따라서 클라 `fnRenderServerError` 는 **SCRIPT 텍스트+타입 우선**, SCRIPT 없는 응답(6·8)만 RTMSG/RETMSG 사용. ②13·14 의 SCRIPT 에 `lf_createMimeCts()` 포함 → **삭제도 CTS 팝업 필요**(원본 삭제 핸들러는 이를 미배선해 전송가능 패키지 삭제 시 무반응/빈메시지였음 → HTML5 는 onCts 배선해 정상화).

### 2.3 검증 결과 (2026-06-30, 전 예외경로 대조)
- 메시지 키 **EN/KO 양쪽 존재 확인**: 050/073/162/193/205/213/305/015/017/092/231/232/003/004/314 → 역현지화 가능.
- **누락 1건**: `/U4A/MSG_WS` **395**(CTS Target System 불일치) 가 로컬 DB에 없음 → 역조회 실패 시 **서버 원문 폴백**(표시는 됨, 클라 언어 재렌더만 불가). 드문 경로라 보류, 추후 키 수집 대상.
- **반영 수정 2건**: (a) `_parseScript` 가 showMessage **타입(E/W/S/I)** 캡처 + `fnRenderServerError` **SCRIPT 우선/타입 보존**. (b) 삭제 경로 **onCts 배선**(CTS 후 재삭제).

## 3. 근본 원인
- 메시지를 **서버 `SY-LANGU`(로그인 언어, 현재 EN)** 로 `MESSAGE ... INTO LV_MSG` 렌더 후 텍스트로 전송.
- 클라는 그 텍스트를 그대로 표시(SCRIPT eval 또는 RETMSG) → **클라 언어 무시**.
- 게다가 SCRIPT 는 `parent.showMessage(sap,...)` 같이 **WS 런타임 전역에 의존** → 별도창 등에서 eval 실패(무반응)까지 유발. (별도창은 전역 안전스텁으로 임시 방어 중 — [[browser-window-common-ux]].)

## 4. 클라이언트 단독 전략 — "서버 텍스트 → 키 역매핑 → 클라 언어 재렌더"
서버를 못 고치므로 응답 형태(RETMSG 텍스트 / SCRIPT 임베드 텍스트 / RETCD)는 그대로 받는다.
클라는 **받은 텍스트로 메시지 키를 역추적**한 뒤 **클라 언어로 다시 렌더**한다. (SCRIPT 는 eval 하지 않고 **파싱**한다.)

### 4.1 단계 (단일 헬퍼 `fnLocalizeServerError(oResult)`)
1. **응답에서 "표시 텍스트 + 후속동작"을 추출**한다.
   - `RETMSG`/`RTMSG`/`MESSAGE` 텍스트가 있으면 그게 표시 텍스트.
   - `SCRIPT` 가 있으면 **eval 하지 말고 파싱**: 정규식으로 `showMessage(..., '<텍스트>')` 의 `<텍스트>` 추출, `lf_createMimeCts` 토큰 있으면 `needCts=true`.
2. **역매핑(text → 키)**: 로컬 메시지 DB로 만든 **역인덱스**에서 표시 텍스트의 메시지 키를 찾는다.
   - 인덱스는 **클라 DB의 전 언어 텍스트**로 구성(서버 렌더 언어를 몰라도 매칭). `정규화(text)` = trim+소문자+공백압축.
   - **무파라미터 메시지**(205/162/004/015/017 등)는 정규화 텍스트 **완전일치**로 키 확정.
   - **파라미터 메시지**(073 `&1 …`, 092 `& …`, 305 `&1 … &2 …`)는 템플릿을 **정규식화**(`&n`→캡처그룹)해 매칭+파라미터 추출. 모호하면 매칭 실패로 간주.
3. **클라 언어 재렌더**: 키를 찾으면 `fnGetMsgClsText(클래스, 번호, 추출파라미터…)` 로 **클라 언어** 메시지 박스(타입 E). 못 찾으면 **받은 서버 텍스트를 그대로**(graceful degrade).
4. **후속동작**: `needCts` 면 CTS 선택 팝업 → 선택 `TRKORR` 로 재시도(eval 없이 직접 호출).
5. 표시할 것이 전혀 없으면 일반 오류(`ZMSG_WS_COMMON_001/314`).

### 4.2 역인덱스 구성 (성능/정확도)
- **범위 한정**: 전체 메시지 DB가 아니라, **분석된 서비스가 실제 쓰는 키 집합만** 인덱싱(과적합/오매칭 방지·경량). 본 문서의 §2.1 표가 그 후보 목록. 새 서비스 분석 시 키를 추가한다.
- 우선 클래스: `/U4A/MSG_WS`(서버 업무 메시지), 필요시 `ZMSG_WS_COMMON_001`.
- 무파라미터 우선 매칭 → 실패 시 파라미터(정규식) 매칭. 둘 다 실패면 원문 표시.

### 4.3 SAP 프레임워크 메시지(키 없음) — 클라 패턴 사전
`INSERT_MIME` 등 프레임워크 메시지(예: 폴더 중복 "… already exists")는 `/U4A/MSG_WS` 키가 아니라 역인덱스에 없다. → **클라가 소형 패턴 사전**을 둔다(언어별 정규식 → 클라 키).
- 예: `/already exists/i` 또는 `/이미 존재/` → `/U4A/MSG_WS` **004**("중복된 파일 이름이 있습니다.").
- 매칭 안 되면 **서버 원문 그대로 표시**(현지화 불가는 어쩔 수 없이 노출, 회귀는 없음).
- ⚠️ 패턴 사전은 **프레임워크 메시지 언어가 서버 설정에 좌우**되므로(EN/KO 둘 다) 양쪽 패턴을 등록한다. 깨지기 쉬우니 **꼭 필요한 대표 케이스만** 최소로.

### 4.4 한계 (명시)
- 역매핑은 **휴리스틱**: 서버 텍스트가 DB 텍스트와 1:1이어야 정확. 서버 메시지가 가공(접두/접미)되면 실패 → 원문 표시로 폴백.
- 파라미터 다수/중간 삽입은 추출이 모호할 수 있음 → 실패 시 원문 표시.
- 프레임워크 메시지는 패턴 사전 등록분만 현지화.
- **완전·정확한 현지화는 서버가 코드(키)를 주는 방식이 정답**이나, 본 전제(서버 불가)에선 위가 최선.

## 5. 구현 / 적용
- **단일 헬퍼**: `oAPP.fn.fnLocalizeServerError(oResult, { onCts })` (별도창은 frame.js, 인앱은 공통 위치). 모든 서버 콜백의 에러 처리를 이걸로 통일.
  - 내부: ①텍스트/needCts 추출(SCRIPT 파싱 포함) ②역인덱스 매핑 ③클라언어 렌더(또는 원문) ④needCts→CTS.
- **역인덱스/패턴사전**: 분석된 서비스(현재 `SET_MIME_CRUD`)의 키만 등록. 키 목록·텍스트는 로컬 DB에서 읽어 런타임 1회 구성.
- **확장**: 새 서비스는 §1 방법으로 ABAP 추적 → 사용하는 `/U4A/MSG_WS` 키를 역인덱스 후보에 추가 + (프레임워크 메시지면) 패턴 사전에 추가.
- **구현 완료(2026-06-29)**: MIME 별도창 `frame.js` `oAPP.fn.fnRenderServerError(oResult,{onCts})` —
  ①SCRIPT **정규식 파싱**(eval 제거: `showMessage(...,'텍스트')` 추출 + `lf_createMimeCts`→needCts)
  ②**역조회**: 메인프로세스 `REMOTE.getGlobal('WsMsgCls').findKeyByText(langu,text)`(완전일치) / `getParamTemplates(langu)`(파라미터 템플릿 정규식 매칭, 언어별 캐시, 리터럴<5 제외)
  ③프레임워크 패턴사전(`already exist|이미 존재/등록` → 004) ④needCts→CTS. 폴더 생성(mime.js)이 소비.
  후보 언어=`["EN","KO"]`, 허용 클래스=`/U4A/MSG_WS`·`ZMSG_WS_COMMON_001`. (메인프로세스 역조회 인프라 `electron/lib/msg/*` 가 이미 존재 — 그대로 활용.)

## 6. 메모
- 메시지 키/파라미터 임의 생성 금지([[no-invented-messages]]) — 역매핑/패턴사전 모두 **로컬 DB의 실제 `/U4A/MSG_WS` 키**만 사용.
- 클라 언어 = Workspace 언어(`getUserInfo().LANGU`) — `fnGetMsgClsText` 가 그 언어로 렌더([[ws-language-mechanism]]).
- 별도창은 `parent` 가 WS 런타임이 아니라 SCRIPT eval 이 실패 → **eval 대신 파싱** 방식이 별도창에서도 견고([[guard-server-script-eval]]).
