| **프로젝트** | U4A Workspace 4.0 — 화면을 최신 웹 기술(HTML5)로 개편 |

---

## 1. 요약

- 기존 화면 제작에 쓰던 **SAP UI5 기술을 걷어내고, 표준 웹 기술(HTML5)로 화면을 다시 만드는** 프로젝트입니다. **6월 12일 착수**하여 현재 **20일차** 진행 중입니다.
- **프로그램의 세 개 주요 화면(WS10 메인 · WS20 디자인 편집 · WS30 소스 편집)의 기본 틀과, 여러 화면에서 공통으로 쓰는 화면 요소(버튼·표·목록·팝업창 등)는 제작 완료**되었습니다. 현재는 개별 팝업창과 세부 기능을 하나씩 옮기는 단계입니다.
- 초기 대비 상당 부분이 완료되었고, **현재 남은 작업은 20건**입니다. 최근 이틀(6/29~30)에만 10여 건을 완료하는 등 후반부 속도가 붙고 있습니다.
- 프로그램의 **기본 구조와 서버 연결 방식은 그대로 두고 화면 부분만 교체**하는 방식이라, 기존 기능이 그대로 유지되며 운영에 영향이 없습니다.

---

## 2. 프로젝트 개요

| 구분 | 내용 |
|---|---|
| **목표** | 오래된 화면 기술(SAP UI5) 제거 → 최신 웹 기술로 교체하여 유지보수 편의성·속도 개선 |
| **범위** | 3대 화면(메인 · 디자인 편집 · 소스 편집)과 딸린 팝업창 전체 |
| **변경 안 함** | 프로그램 내부 엔진, 서버 통신 방식, 파일 저장 방식 (그대로 유지) |
| **방식** | 화면을 하나씩 순서대로 교체 + 공통으로 쓰는 화면 요소를 먼저 표준으로 만들어 재사용 |

## 5. 향후 계획

남은 20건을 중요도·규모에 따라 **3단계**로 나누어 진행합니다.

| 단계 | 대상 | 주요 내용 | 목표 |
|:--:|---|---|---|
| **1단계** (우선) | 데이터 연결 팝업 + 디자인 편집 세부 기능 | 사용 빈도가 가장 높은 **데이터 연결 팝업**(규모 최대)을 먼저 교체하고, 디자인 편집 화면의 세부 기능(화면요소 ID 변경·미리보기 연동 등)을 마무리 | 핵심 기능 안정화 |
| **2단계** | 나머지 팝업창 (약 10종) | 아이콘 목록·소스 패턴·업데이트 안내·바로가기 생성 등 개별 팝업창을 순차 교체 (표준 틀 재사용으로 속도 확보) | 팝업창 전부 교체 완료 |
| **3단계** | 소스 편집기 부가 기능 + 마무리 | 소스 편집기 우클릭 메뉴의 실행 기능(패턴 삽입·디자이너 등) 연결, 전체 화면 점검·안정화 | 전 화면 교체 완료 |

**추진 목표 일정**

| 구분 | 목표 시점 | 비고 |
|---|---|---|
| **HTML 화면 전환 완료** | **7월 중순** (약 2주 내) | 남은 20건 완료 목표 |
| **기능 테스트(QA)** | **~8월 초** (약 2~3주) | 화면별 동작 확인 + 통합 점검 |
| **안정화·오픈 준비** | **8월 중순** | 테스트 보완 후 오픈 |

- **7월 중순까지 화면 전환을 완료**하고, 이어 기능 테스트를 거쳐 **8월 중순 오픈을 목표**로 추진합니다.
- 기능 테스트는 전환된 화면을 실제로 조작해 기존 UI5 화면과 **동작이 동일한지 확인**하는 과정으로, 화면 수가 많아 별도 기간을 확보했습니다.
- 서버·프로그램 기본 구조는 변경하지 않으므로, 진행 중에도 **운영·기존 기능에 영향은 없습니다.**
